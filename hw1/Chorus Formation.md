# Chorus Formation/合唱队形
> https://www.luogu.com.cn/problem/P1091

这道题是经典的动态规划（DP）问题，通常被称为 **“合唱队形”**。下面我为你详细拆解题目的含义以及输入输出的规则。

---

### 1. 题目含义解析

**核心目标：**
通过删掉最少的人，让剩下的人排成一个“先升后降”的队形（即：身高先从小到大递增，达到某一个人后，再从大到小递减）。

**“合唱队形”的定义：**
假设剩下的学生身高序列为 $t_1, t_2, \dots, t_k$，必须存在一个峰值点 $i$（$1 \le i \le k$），使得：
*   从左往右看：$t_1 < t_2 < \dots < t_i$ （身高递增）
*   从峰值点往右看：$t_i > t_{i+1} > \dots > t_k$ （身高递减）

**为什么是“最少删减人数”？**
由于题目要求删掉的人最少，这等价于我们需要找到一个**最长的合唱队形**（即剩下的 $k$ 尽可能大），然后用总人数 $n$ 减去剩下的最大人数 $k$，得到的就是最少需要删掉的人数。

---

### 2. 输入输出解释

#### 输入格式 (Input)
*   **第一行**：一个整数 $n$（表示总共有多少名学生，范围 2 到 100）。
*   **第二行**：$n$ 个整数（表示每个学生的身高，身高在 130 到 230 之间）。

#### 输出格式 (Output)
*   一个整数，代表**最少需要走掉的人数**。

---

### 3. 如何解题（算法思路提示）

要找到最长的合唱队形，你可以将问题拆分为两个部分：

1.  **左侧最长上升子序列 (LIS)**：
    对于每个位置 $i$，计算以 $i$ 结尾、向左看的最长上升序列长度（使用动态规划）。
2.  **右侧最长下降子序列 (LDS)**：
    对于每个位置 $i$，计算以 $i$ 开始、向右看的最长下降序列长度（或者说，从右往左看的最长上升序列）。
3.  **合并计算**：
    合唱队形的长度等于：`以 i 为中心的最长上升序列长度` + `以 i 为中心的最长下降序列长度` - 1（因为 $i$ 本身被计算了两次）。
4.  **最终答案**：
    遍历所有可能的 $i$ 作为峰值点，找出其中最大的合唱队形长度 $k_{max}$，最后输出 $n - k_{max}$。

---

### 4. 样例分析 (Sample #1)

*   **输入：** `8` 名学生，身高分别为 `186 186 150 200 160 130 197 220`
*   **分析：**
    *   如果我们保留 `150, 160, 130` 这种形状是行不通的（因为中间不能平）。
    *   一个可行的合唱队形例子：`186, 200, 160, 130` (长度为4)。
    *   总人数 $8 - 4 = 4$，所以最少要走掉 4 个人。

**总结：** 这是一道**求最长双调子序列（Bitonic Subsequence）**的问题，利用动态规划可以有效地解决。如果你在编写代码时遇到困难，可以随时问我！

## 代码实现细节
你的直觉其实很准！你提到的“和左右相比”恰好就是 DP（动态规划）的核心：**一个位置的最优解，是由它前面的位置（或者后面的位置）的状态推导出来的。**

### 为什么用 DP？
因为我们要计算“以某个人为中心，左边有多少人递增，右边有多少人递减”。
对于位置 `i`，它能作为峰值，取决于：
1. **左边：** 以前面的某个人 `j` 为结尾的最长上升子序列长度。
2. **右边：** 以后面的某个人 `k` 为开始的最长下降子序列长度。

这就是 **最优子结构**：求 `i` 的状态，需要先算出比 `i` 小的所有 `j` 的状态。

---

### 如何实现？

我们可以拆解成两个数组：
1. `f[i]`：表示从左往右看，以第 `i` 个人结尾的最长上升子序列长度。
2. `g[i]`：表示从右往左看，以第 `i` 个人结尾的最长上升子序列长度（这其实就是以 `i` 为起点的最长下降子序列）。

#### 1. 计算 `f[i]` (左往右上升)
对于每一个 `i`，遍历所有的 `j < i`：
如果 `line[i] > line[j]`，说明 `i` 可以排在 `j` 后面，那么 `f[i] = max(f[i], f[j] + 1)`。

#### 2. 计算 `g[i]` (右往左上升 / 即从 `i` 开始递减)
同理，从右向左遍历，对于每一个 `i`，遍历所有的 `j > i`：
如果 `line[i] > line[j]`，说明 `i` 可以排在 `j` 前面，那么 `g[i] = max(g[i], g[j] + 1)`。

---

### 示例代码实现

```cpp
#include <iostream>
#include <vector>
#include <algorithm>
using namespace std;

int main() {
    int n;
    cin >> n;
    vector<int> line(n);
    for (int i = 0; i < n; i++) cin >> line[i];

    // f[i] 存以 i 结尾的最长上升子序列
    vector<int> f(n, 1);
    for (int i = 0; i < n; i++) {
        for (int j = 0; j < i; j++) {
            if (line[i] > line[j]) {
                f[i] = max(f[i], f[j] + 1);
            }
        }
    }

    // g[i] 存以 i 开始的最长下降子序列 (从右往左看即上升)
    vector<int> g(n, 1);
    for (int i = n - 1; i >= 0; i--) {
        for (int j = n - 1; j > i; j--) {
            if (line[i] > line[j]) {
                g[i] = max(g[i], g[j] + 1);
            }
        }
    }

    // 计算最大合唱队形长度
    int max_k = 0;
    for (int i = 0; i < n; i++) {
        // -1 是因为 i 位置的人在 f 和 g 中都被计算了一次
        max_k = max(max_k, f[i] + g[i] - 1);
    }

    cout << n - max_k << endl;
    return 0;
}
```

### 关键点总结：
*   **为什么要减 1？** 因为 `f[i]` 包含了 `i` 本身，`g[i]` 也包含了 `i` 本身，加起来时 `i` 被重复加了一次，所以要减掉。
*   **复杂度：** $O(n^2)$。由于题目要求 $n \le 100$，这个复杂度完全足够，甚至比 $O(n \log n)$ 的解法更直观。
*   **逻辑流程：** 先计算全队的上升趋势 -> 再计算全队的下降趋势 -> 最后找一个人作为“中间人”把左右拼起来 -> 剩下的就是我们要删除的人数。
