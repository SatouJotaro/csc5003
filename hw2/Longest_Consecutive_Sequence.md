# Longest Consecutive Sequence/最长连续序列 medium

## 1. 题目回顾（中文）

**输入**  
- 第 1 行：一个整数 `n`（`0 ≤ n ≤ 10^5`），表示数组长度。  
- 第 2 行（若 `n>0`）：`n` 个整数 `nums[i]`（`-10^9 ≤ nums[i] ≤ 10^9`），用空格分隔。  
- 当 `n = 0` 时，第二行可以不存在。

**输出**  
- 只输出一个整数：`nums` 中最长的 **连续整数序列** 的长度。

> **连续整数序列** 的定义：  
> - 序列中的相邻两个数相差恰好为 `1`（如 `1,2,3,4`）。  
> - 序列中的元素在原数组中可以不相邻，只要满足数值上的连续即可。  
> - 同一个数值出现多次时只计一次（因为我们关心的是“连续整数的集合”，不是出现次数）。

**示例**  

```
输入 #1
6
100 4 200 1 3 2
输出 #1
4
解释：最长的连续序列是 1,2,3,4，长度为 4

输入 #2
10
0 3 7 2 5 8 4 6 0 1
输出 #2
9
解释：连续序列 0,1,2,3,4,5,6,7,8 长度 9（注意 0 出现了两次，但仍只计一次）
```

---

## 2. 涉及的知识点

1. **哈希集合（Hash Set）**  
   - 查找、插入、平均时间复杂度 O(1)。在本题中用于快速判断某个数是否出现。  

2. **一次性遍历 + 起点判定**  
   - 只对“序列的起点”（即 `x-1` 不在集合中）进行完整计数，避免对同一个序列的每个元素都重复检查，从而把整体时间压到 **O(n)**。

3. **空间换时间**  
   - 用额外的哈希集合保存全部元素，空间 O(n)。

---

## 3. 解题思路（一步步细化）

### 3.1 关键观察

- 如果我们遍历数组的每一个元素 `x`，并且从 `x` 开始向后不断 `x+1, x+2…` 检查它们是否在集合里，就会产生大量重复计数（比如序列 `1,2,3,4` 会在 `1、2、3、4` 四次被全部检查）。  
- 为了避免这种重复，只在 **`x-1` 不在集合中** 时，才把 `x` 当作序列的“起点”，从此开始往后计数。这样每个序列只会被完整遍历一次。

### 3.2 步骤

1. **把所有元素放进哈希集合**（`unordered_set<int>` 或 `unordered_set<long long>`）。  
2. 初始化 `ans = 0`（记录最大长度）。  
3. 对集合中的每个元素 `x` 执行：  
   a. 如果 `x-1` **不在** 集合中，说明 `x` 是某个连续序列的起点。  
   b. 从 `x` 开始，依次检查 `cur = x, x+1, x+2 …`，每出现一次就把计数器 `cnt` 加 1，直到 `cur+1` 不在集合为止。  
   c. 更新 `ans = max(ans, cnt)`。  
4. 最后输出 `ans`（如果 `n==0`，ans 保持为 `0`）。

### 3.3 为什么是 O(n)？

- 每个元素最多被放进集合一次（O(n)）。
- 对于每个起点 `x`，我们遍历 `x, x+1, …`，而这些数字恰好是该连续序列的全部成员。整个数组的每个元素 **只会在一次计数过程** 中被访问一次（因为一旦它是序列成员，它不可能再成为另一个序列的起点——它的前驱 `x-1` 必然在集合里）。  
- 因此整体时间复杂度是 **O(n)**，空间复杂度是 **O(n)**（存放集合）。

---

## 4. 代码实现（C++17）

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int n;
    if (!(cin >> n)) return 0;          // 防止空输入
    vector<long long> a(n);
    for (int i = 0; i < n; ++i) cin >> a[i];
    
    // 1. 把所有元素放进 unordered_set
    unordered_set<long long> st;
    st.reserve(n * 2);   // 提前分配，减少 rehash
    for (auto v : a) st.insert(v);
    
    int longest = 0;
    // 2. 逐个检查是否为起点
    for (const auto &x : st) {
        // 只有当 x-1 不在集合里时，x 才是起点
        if (st.find(x - 1) == st.end()) {
            int cnt = 0;
            long long cur = x;
            // 3. 持续向 x+1 方向延伸
            while (st.find(cur) != st.end()) {
                ++cnt;
                ++cur;
            }
            longest = max(longest, cnt);
        }
    }
    
    cout << longest << "\n";
    return 0;
}
```

### 代码要点说明

| 步骤 | 代码片段 | 解释 |
|------|----------|------|
| **读取输入** | `int n; if(!(cin>>n))return 0;` | 兼容 `n==0` 的情况，直接结束。 |
| **存入集合** | `unordered_set<long long> st;` | 使用 `long long`（或 `int`）均可，因为值范围在 `[-10^9,10^9]`。`reserve` 提升性能。 |
| **起点判定** | `if (st.find(x-1) == st.end())` | `x-1` 不在集合里 → `x` 为序列起点。 |
| **向后延伸计数** | `while (st.find(cur) != st.end()) { ++cnt; ++cur; }` | 只要 `cur` 存在，就把计数 `cnt` 加 1 并继续检查 `cur+1`。 |
| **更新答案** | `longest = max(longest, cnt);` | 记录最长长度。 |
| **输出** | `cout << longest << '\n';` | 最终结果。 |

#### 为什么使用 `unordered_set` 而不是 `std::set`？

- `unordered_set` 的查找/插入均摊 O(1)，而 `std::set`（红黑树）O(log n)。对本题 `O(n log n)` 也能通过（因为 n ≤ 1e5），但 `unordered_set` 能把时间压到线性，更符合题目提示。

#### 特殊情况的处理

- **n = 0** → 循环根本不会进入，`longest` 保持 `0`，输出 `0` 符合预期。  
- **全是相同的数** → 集合中只有一个元素，起点判定会成立一次，计数得到 `1`。  
- **负数** → `x-1` 计算不会出现溢出（因为 `long long` 足够大），若担心可使用 `std::numeric_limits<long long>::min()` 边界检查，这里不必。

## 6. 其他思路（供参考）

- **排序后一次遍历**：把数组排序（`O(n log n)`），然后线性扫描，记录当前连续段长度并取最大。  
  - 优点：代码更直观。  
  - 缺点：时间复杂度比哈希方法稍高（`O(n log n)`），但仍满足题目约束。  
- **哈希 + 记忆化 DP**：利用 `unordered_map<int,int>` 保存已经计算过的序列长度（记忆化），同样可以做到 O(n)。  
  - 这里不展开，因为题目提示的“起点判定”已经足够简洁。

---

## 7. 小结

- **核心思想**：利用哈希集合快速查找，只从“序列的起点”开始计数，确保每个连续整数序列只遍历一次。  
- **实现要点**：  
  1. 把所有元素放进 `unordered_set`。  
  2. 对每个元素 `x`，如果 `x-1` 不在集合里，则 `x` 是起点。  
  3. 计数 `x, x+1, x+2…` 的长度，更新答案。  
- **时间/空间**：均为 O(n)，满足 `n ≤ 10^5` 的限制。

## 1. 为什么需要 `unordered_set`？

### 1.1 哈希表是什么？

想象你有一本**字典**，要查一个汉字"龙"，你直接翻到"L"那几页就能快速找到——这就是**哈希表**的核心思想。

```
数组: [100, 4, 200, 1, 3, 2]

如果我想知道 "1 在不在数组里"：
- 遍历数组检查 → O(n) 时间，可能要查 6 次
- 用哈希表 → O(1) 时间，直接定位
```

### 1.2 `unordered_set` vs 普通数组

| 普通数组 | `unordered_set`（哈希表） |
|---------|------------------------|
| 查找某个值是否存在，需要遍历整个数组 | 查找直接定位，平均 O(1) |
| `[1,2,2,3]` 有重复元素 | 自动去重，只存唯一的元素 |
| `nums[0]` 这种随机访问 | `set.count(x)` 或 `set.find(x)` 检查 |

```
数组 [1,2,2,3] 放进 unordered_set 后变成 {1,2,3}
```

### 1.3 代码解释

```cpp
// 这是创建空的哈希表
unordered_set<long long> st;

// 这是把数组元素一个个"倒"进哈希表
for (auto v : a) {
    st.insert(v);  // 自动去重，查找快
}
```

## 2. `reserve()` 和 `rehash` 详解

### 2.1 什么是 rehash？

哈希表内部有一个**桶数组**（bucket array），用来存放元素。

```
初始状态（假设有 8 个桶）：
  桶0  桶1  桶2  桶3  桶4  桶5  桶6  桶7
  [ ]  [ ]  [ ]  [ ]  [ ]  [ ]  [ ]  [ ]

插入 5 个元素后：
  桶0  桶1  桶2  桶3  桶4  桶5  桶6  桶7
  [5]  [1]  [6]  [2]  [ ]  [ ]  [7]  [3]
```

当元素太多时（桶快满了），哈希表会**重新分配更多桶**，把所有元素重新分配位置——这就是 **rehash**。

```
rehash 后（变成 16 个桶）：
  桶0  桶1  桶2  桶3  桶4  桶5  桶6  桶7 ...
  [ ]  [1]  [2]  [3]  [5]  [6]  [7]  [ ] ...
```

### 2.2 为什么要减少 rehash？

- **rehash 很慢**：要遍历所有元素，重新计算哈希值，重新分配位置
- **如果提前分配好足够的空间**，就不用 rehash，性能更好

### 2.3 `reserve(n*2)` 的含义

```cpp
st.reserve(n * 2);  // 预分配能容纳 n*2 个元素的空间
```

```
n = 6 个元素
reserve(12) = 预先准备 12 个桶（大约）

这样插入 6 个元素时，负载因子 = 6/12 = 0.5
不需要 rehash，性能更好
```

**注意**：`reserve` 分配的是"桶的数量"，不是确切存多少元素，而是预留空间让 rehash 不会发生。

## 3. 为什么先存数组 `a`，再存 `st`？

### 两种写法对比

## 5. `st.find(x - 1) == st.end()` 解释

### 5.1 `find()` 方法

```cpp
auto it = st.find(x);  // 在哈希表中查找 x

if (it == st.end()) {
    // 没找到！
} else {
    // 找到了！it 指向这个元素
}
```

### 5.2 `end()` 是什么？

```
unordered_set 的内部结构：

[桶0] → [元素1] → [元素2]
[桶1] → [元素3]
[桶2] → 空
[桶3] → [元素4]
...
[桶N] → 空

st.begin() 指向第一个元素
st.end()   指向"末尾的下一个位置"（一个哨兵，表示"没有更多元素了"）
```

```
找得到的情况：
  st.find(5) → 指向 5 的位置 → 不等于 st.end() → 找到了

找不到的情况：
  st.find(999) → 找不到 → 返回 st.end() → 等于 st.end()
```

### 5.3 整句代码

```cpp
if (st.find(x - 1) == st.end()) {
    // x-1 不存在 → x 是起点
}
```

```
举例：st = {1,2,3,4,100,200}

x = 1 时：
  find(0) == end()?   → 0 不在集合里 → true → 1 是起点 ✓

x = 2 时：
  find(1) == end()?   → 1 在集合里 → false → 2 不是起点 ✗

x = 100 时：
  find(99) == end()?  → 99 不在集合里 → true → 100 是起点 ✓
```

## 6. 完整代码加注释（逐行解释）

```cpp
#include <bits/stdc++.h>      // 包含所有标准库（方便）
using namespace std;          // 不用写 std::

int main() {
    ios::sync_with_stdio(false);  // 加速 I/O
    cin.tie(nullptr);             // 加速 I/O
    
    int n;
    if (!(cin >> n)) return 0;    // 读 n，如果读不到就结束
    
    vector<long long> a(n);       // 开一个数组存输入
    for (int i = 0; i < n; ++i)   // 逐个读入
        cin >> a[i];
    
    // ========== 哈希表部分 ==========
    unordered_set<long long> st;  // 创建空哈希表
    st.reserve(n * 2);            // 预分配空间，减少 rehash
    
    for (auto v : a)              // 遍历数组 a
        st.insert(v);             // 把每个元素放进哈希表（自动去重）
    // ========== 哈希表部分结束 ==========
    
    int longest = 0;              // 答案，初始为 0
    
    // 遍历哈希表中的每个元素
    for (const auto &x : st) {    // x 是 st 中的一个值
    
        // 关键判断：x-1 不存在 → x 是序列起点
        if (st.find(x - 1) == st.end()) {
            
            int cnt = 0;          // 当前序列长度
            long long cur = x;    // 从 x 开始往后数
            
            // 只要 cur 在集合里，就继续
            while (st.find(cur) != st.end()) {
                ++cnt;            // 长度 +1
                ++cur;            // 下一个数
            }
            
            longest = max(longest, cnt);  // 更新最大长度
        }
    }
    
    cout << longest << "\n";      // 输出答案
    return 0;
}
```

# 哈希表如何自动去重？

你说得对！让我详细解释 `insert` 背后发生了什么。

---

## 1. 插入过程的真相

当调用 `st.insert(v)` 时，哈希表内部是这样工作的：

```cpp
// unordered_set::insert 的内部逻辑（简化版）
template<typename Key>
void unordered_set<Key>::insert(const Key& value) {
    
    // 步骤1：用哈希函数计算位置
    size_t index = hash_function(value) % bucket_count;
    
    // 步骤2：检查该桶里是否已经有这个值
    for (auto& element : bucket[index]) {
        if (element == value) {
            // 发现重复！直接返回，不插入
            return;
        }
    }
    
    // 步骤3：没找到重复，真正插入
    bucket[index].push_back(value);
    ++size;
}
```

## 2. 直观比喻

想象一个**图书馆的图书管理系统**：

```
哈希函数 = 按书名首字母分类

桶0(A)  桶1(B)  桶2(C)  桶3(D)  ...
  ↓       ↓       ↓       ↓
[阿婆]  [呗]    [陈]    [董]

现在要插入一本叫"陈"的书：
1. 计算哈希 → 归到"桶2(C)"
2. 桶2已经有"陈"了吗？ → 有！
3. → 不插入，重复了
```

**关键**：插入前会**先检查**，如果已有就**不插**。

## 3. 代码验证

```cpp

**运行结果**：

```

可以看到：
- 重复插入不会增加 `size`
- 最终集合只有唯一元素

## 4. `insert` 的返回值

`insert` 还返回是否真正插入了新元素：

```cpp
#include <iostream>
#include <unordered_set>
using namespace std;

int main() {
    unordered_set<int> st;
    
    // insert 返回 pair<iterator, bool>
    // bool = true 表示是新插入，false 表示已存在
    auto result1 = st.insert(1);
    cout << "插入 1: " << (result1.second ? "新插入" : "已存在") << endl;
    
    auto result2 = st.insert(1);
    cout << "再次插入 1: " << (result2.second ? "新插入" : "已存在") << endl;
    
    auto result3 = st.insert(2);
    cout << "插入 2: " << (result3.second ? "新插入" : "已存在") << endl;
    
    return 0;
}
```

**运行结果**：
```
插入 1: 新插入
再次插入 1: 已存在
插入 2: 新插入
```

## 6. 总结

| 问题 | 答案 |
|-----|------|
| `insert` 如何去重？ | 插入前**先查找**，发现重复就**不插入** |
| 重复插入后 `size` 变吗？ | **不变** |
| 如何判断是否是新插入？ | 看 `insert` 返回值的 `bool` 部分 |
| 这是哈希表特有的吗？ | 是的，`unordered_set`、`unordered_map`、`std::set` 都有去重机制 |

**简单记**：`insert` = "有就不插，没有才插" 😊

# 1. 不同数据结构的 `insert` 对比

并不是所有数据结构的 `insert` 都去重！看对比：

| 数据结构 | `insert` 行为 | 去重吗？ |
|---------|--------------|---------|
| `vector` | 插在末尾 | ❌ 不去重，重复元素都保留 |
| `list` | 插在指定位置 | ❌ 不去重 |
| `deque` | 插在头或尾 | ❌ 不去重 |
| `set` | 插入到正确位置 | ✅ 自动去重 |
| `unordered_set` | 插入到哈希桶 | ✅ 自动去重 |
| `map` | 按 key 插入 | ✅ key 自动去重 |

## 代码验证

```cpp
#include <iostream>
#include <vector>
#include <list>
#include <set>
#include <unordered_set>
using namespace std;

int main() {
    // ====== vector：不去重 ======
    vector<int> v;
    v.insert(v.end(), 1);  // 插末尾
    v.insert(v.end(), 1);  // 又插一个 1
    v.insert(v.end(), 2);
    cout << "vector: "; 
    for (int x : v) cout << x << " ";  // 输出: 1 1 2
    cout << " | size = " << v.size() << endl;

    // ====== list：不去重 ======
    list<int> lst;
    lst.insert(lst.end(), 1);
    lst.insert(lst.end(), 1);
    lst.insert(lst.end(), 2);
    cout << "list:   "; 
    for (int x : lst) cout << x << " ";  // 输出: 1 1 2
    cout << " | size = " << lst.size() << endl;

    // ====== set：自动去重 ======
    set<int> s;
    s.insert(1);
    s.insert(1);  // 重复，被忽略
    s.insert(2);
    cout << "set:    "; 
    for (int x : s) cout << x << " ";  // 输出: 1 2
    cout << " | size = " << s.size() << endl;

    // ====== unordered_set：自动去重 ======
    unordered_set<int> us;
    us.insert(1);
    us.insert(1);  // 重复，被忽略
    us.insert(2);
    cout << "unorder_set: "; 
    for (int x : us) cout << x << " ";  // 输出可能是: 2 1（无序）
    cout << " | size = " << us.size() << endl;

    return 0;
}
```

**运行结果**：
```
vector: 1 1 2  | size = 3
list:   1 1 2  | size = 3
set:    1 2    | size = 2
unorder_set: 2 1  | size = 2
```

## 2. 什么是红黑树？

### 2.1 二叉搜索树（BST）

首先理解**二叉搜索树**：

```
       5
      / \
     3   7
    / \   \
   1   4   9

特点：左子树 < 根节点 < 右子树
```

**查找过程**（找 4）：
```
5 ≠ 4，向左走
3 ≠ 4，向右走
4 = 4，找到了！
```

如果树是**平衡**的，查找只需要 O(log n) 步。

---

### 2.2 树会"退化"！

如果插入的数据是有序的，树会退化成链表：

```
按 1, 2, 3, 4, 5, 6, 7 插入：

       1
        \
         2
          \
           3
            \
             4 ...（退化成链表！）

查找 7 需要 7 步 = O(n)，失去二叉树的优势
```

---

### 2.3 红黑树：自动保持平衡

**红黑树**是一种"自平衡二叉搜索树"，通过给节点涂**红/黑颜色**来保持平衡：

```
红黑树例子：

       5(黑)
      /     \
   2(红)    7(红)
   /  \     /  \
1(黑)3(黑)6(黑)8(黑)
```

**红黑树的规则**：
1. 节点是红色或黑色
2. 根节点必须是黑色
3. 红色节点的子节点必须是黑色（不能连续两个红）
4. **从根到叶子的每条路径，黑色节点数相同**

**平衡原理**：
- 当树要"歪"的时候，通过**旋转 + 变色**自动调整
- 始终保持大致平衡，查找效率稳定在 O(log n)

---

### 2.4 插入过程演示

按 `1, 2, 3, 4, 5` 插入到红黑树：

```
插入 1：                    插入 2：
    1(黑)                      1(黑)
                               \
                              2(红)

插入 3：                    旋转后：
    1(黑)                        2(黑)
   /   \                       /   \
(红)2(红)  (红)3            1(红)  3(红)
                                ↓
                           自动调整，保持平衡！
```

**关键**：红黑树会**自动旋转变色**，永远不让树退化成链表！

---

### 2.5 红黑树 vs 哈希表

| 特性 | 红黑树 (`set`) | 哈希表 (`unordered_set`) |
|-----|---------------|------------------------|
| 实现方式 | 平衡二叉树 | 哈希桶数组 |
| 查找时间 | O(log n) | O(1) 平均 |
| 插入时间 | O(log n) | O(1) 平均 |
| **有序** | ✅ 自动排序 | ❌ 无序 |
| 最坏情况 | O(log n) | O(n)（大量哈希冲突） |
| 内存开销 | 每个节点存颜色 + 指针 | 可能有空桶，负载因子 |

```
set 输出有序：   1 2 3 4 5 6 7
unordered_set： 7 3 1 5 2 6 4（顺序不确定）
```

## 3. 什么时候用什么？

| 场景 | 推荐数据结构 |
|-----|-------------|
| 需要快速查找 | `unordered_set`（O(1)） |
| 需要有序输出 | `set`（O(log n)） |
| 只需要存一堆数据，允许重复 | `vector` |
| 需要频繁在中间插入/删除 | `list` |
| 需要按键值对查找 | `map` / `unordered_map` |

**记住**：`set` 和 `unordered_set` 都去重，但 `set` 有序，`unordered_set` 更快！ 😊
