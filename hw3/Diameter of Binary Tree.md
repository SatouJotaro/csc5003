# Diameter of Binary Tree/二叉树的直径(easy)

## 核心概念

**二叉树的直径** = 树中任意两节点之间最长路径的**边数**（不一定经过根节点）

---

## 输入格式解析

```
5          ← n=5，共5个节点（编号0~4）

1 1 2      ← 节点0：值=1，左孩子=节点1，右孩子=节点2
2 3 4      ← 节点1：值=2，左孩子=节点3，右孩子=节点4
3 -1 -1    ← 节点2：值=3，无左孩子，无右孩子
4 -1 -1    ← 节点3：值=4，无左孩子，无右孩子
5 -1 -1    ← 节点4：值=5，无左孩子，无右孩子
```

> **注意**：`left_i / right_i` 是**子节点的索引号**，不是值！`-1` 表示不存在

---

## 建树可视化

```
         节点0 (val=1)
        /             \
   节点1 (val=2)    节点2 (val=3)
   /         \
节点3(val=4)  节点4(val=5)
```

---

## 样例分析

找所有路径中最长的：

| 路径 | 经过边数 |
|------|---------|
| 节点3 → 节点1 → 节点0 | 2 |
| 节点3 → 节点1 → 节点0 → 节点2 | **3** ✅ |
| 节点4 → 节点1 → 节点0 → 节点2 | **3** ✅ |
| 节点3 → 节点1 → 节点4 | 2 |

最长路径有 **3条边** → 输出 `3`

---

## 算法思路

```
对每个节点，直径候选值 = 左子树深度 + 右子树深度

diameter(node) = max(
    左子树深度 + 右子树深度,   ← 路径经过当前节点
    diameter(左子树),          ← 路径在左子树内部
    diameter(右子树)           ← 路径在右子树内部
)
```

# 涉及的算法与数据结构知识点

## 🗂️ 数据结构

### 1. 二叉树 (Binary Tree)
```
- 节点结构：值 + 左孩子索引 + 右孩子索引
- 用数组存储节点（按索引随机访问）
- 根节点固定为 index = 0
```

### 2. 递归调用栈 (Implicit Stack)
```
- 递归本质上使用系统栈
- 树的深度决定栈的深度
- 最坏情况（链状树）：O(n) 栈深度
```

---

## ⚙️ 算法

### 1. 深度优先搜索 DFS ⭐核心
```
后序遍历（Post-order）：
  先处理左子树 → 再处理右子树 → 最后处理当前节点

为什么用后序？
  计算当前节点的深度，必须先知道子树的深度
```

### 2. 树形DP (Tree DP) ⭐核心
```
定义子问题：
  depth(node) = 以该节点为根的子树最大深度

状态转移：
  depth(node) = max(depth(左孩子), depth(右孩子)) + 1

全局维护答案：
  ans = max(ans, depth(左孩子) + depth(右孩子))
```

### 3. 递归 (Recursion)
```
递归三要素：
  ① 终止条件：node == -1，返回 -1（或0）
  ② 递归关系：depth = max(left, right) + 1
  ③ 返回值：当前子树的最大深度
```

---

## 📊 复杂度分析

| 维度 | 复杂度 | 原因 |
|------|--------|------|
| 时间 | **O(n)** | 每个节点只访问一次 |
| 空间 | **O(n)** 最坏 | 递归栈深度（链状树）|
| 空间 | **O(log n)** 最好 | 完全平衡二叉树 |


# C++ 解题代码

## 完整代码

```cpp
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 100005;

int val[MAXN];   // 节点的值
int lc[MAXN];    // 左孩子索引
int rc[MAXN];    // 右孩子索引
int n;
int ans = 0;     // 全局最大直径

/*
 * DFS 后序遍历
 * 返回值：以 node 为根的子树深度（边数）
 *   node == -1 时返回 -1（空节点，贡献0条边）
 *   叶子节点返回 0
 */
int dfs(int node) {
    // 空节点，返回 -1
    if (node == -1) return -1;

    // 后序：先递归左右子树
    int left_d  = dfs(lc[node]);
    int right_d = dfs(rc[node]);

    // 经过当前节点的路径长度：
    //   left_d  + 1 = 当前节点 到 左子树最远叶子 的边数
    //   right_d + 1 = 当前节点 到 右子树最远叶子 的边数
    // 若某侧为空（返回-1），则 -1+1=0，即贡献0条边 ✓
    ans = max(ans, (left_d + 1) + (right_d + 1));

    // 返回当前子树深度（取较深一侧 +1）
    return max(left_d, right_d) + 1;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    cin >> n;

    // 读入所有节点
    for (int i = 0; i < n; i++) {
        cin >> val[i] >> lc[i] >> rc[i];
    }

    // 特判：n <= 1 直径为 0
    if (n <= 1) {
        cout << 0 << endl;
        return 0;
    }

    // 从根节点（索引0）开始DFS
    dfs(0);

    cout << ans << endl;

    return 0;
}
```

## 代码细节
在 ACM/ICPC 竞赛编程中，使用全局静态数组代替动态数组（如 `std::vector` 或 `new`）是一种非常经典的工程策略。以下是背后的深层原因：

### 1. 性能优化（最重要的原因）
*   **内存分配开销：** `std::vector` 或动态分配对象（`new/malloc`）在运行时需要向操作系统申请内存。如果在递归过程中频繁创建或销毁节点对象，分配器的开销会显著增加，甚至导致超时（TLE）。
*   **缓存友好性（Cache Locality）：** 静态数组分配在程序的 `.data` 或 `.bss` 段，内存是**连续的**。当 CPU 从内存读取数据时，连续的内存能利用 CPU 的**缓存预取（Cache Prefetching）**机制，访问速度远快于分散在堆空间里的零碎对象。

### 2. 避免内存碎片与管理成本
*   当数据量极大（如 $10^5$ 到 $10^6$ 个节点）时，频繁的内存分配可能导致内存碎片。
*   使用静态数组后，**不需要写析构函数或手动 `delete`**，完全避免了内存泄漏的风险。在竞赛环境下，只要代码能跑出结果，程序退出前内存释放慢点甚至不释放都是被允许的。

### 3. 程序稳定性（“稳”字当头）
*   **栈空间溢出：** 如果用递归 + 动态分配对象，可能会因为析构函数递归调用产生额外的栈开销。静态数组直接放在全局区，**不占用栈空间**，这使得程序能处理更深的树（甚至递归层数接近 $10^5$ 规模的链状树）。
*   **简单直接：** 竞赛的时间非常紧迫。使用下标 `lc[i]` 和 `rc[i]` 访问比使用指针 `node->left->val` 更不容易产生空指针解引用（Null Pointer Dereference）的 Bug，调试也更容易。

### 4. ACM/OJ 平台的运行环境
*   **内存分配限制：** 在某些 OJ 平台，对程序运行时的内存分配有严格限制。全局静态数组在编译时就确定了占用空间，提交上去后平台可以准确评估内存使用，不会因为动态申请内存导致的空间抖动。
*   **快速初始化：** 静态数组在程序启动时由操作系统一次性清零，`val`、`lc` 等数组自动变成 0，非常方便。

### 什么时候改用动态结构？
虽然全局数组在竞赛中是“王者”，但在实际工程开发（如企业级项目）中，通常推荐使用 `std::vector` 或智能指针：

1.  **数据规模不确定：** 如果题目没给 `n` 的范围，或 `n` 非常巨大，静态数组会导致“内存超限（MLE）”，此时只能用 `std::vector` 或动态链表。
2.  **代码可读性与封装：** 工程代码更注重模块化，使用类（`class/struct`）封装节点信息更能反映逻辑结构，方便维护和复用。

### 总结
在 ACM 题目中：
*   **对于 $N \le 10^5$ 的题目**：全局静态数组是**最优选择**，省心、极快、防溢出。
*   **对于内存要求极高的题目**：如果全局数组 MLE，才会考虑使用链式前向星或者其他压缩存储方式，或者在不得已时才用动态分配。

**一句话总结：** 在竞赛里，**“静态化”和“数组化”**是提升代码性能和稳定性的“金律”。

## DFS, BFS, 前序，后序
### 1. 什么是 DFS？
**DFS (Depth First Search，深度优先搜索)**，顾名思义就是“**一条路走到黑**”。
当你到达一个节点，先不管别的，直接深入它的子分支，直到无路可走（叶子节点），再回溯（退回到前一个节点），看看还有没有别的路没走。

**DFS 的通用模板：**
```cpp
void dfs(int node) {
    if (node == -1) return; // 终止条件：空节点
    
    // 1. 前序位置（刚刚进入节点）
    
    dfs(left[node]);  // 深入左子树
    
    // 2. 中序位置
    
    dfs(right[node]); // 深入右子树
    
    // 3. 后序位置（准备离开节点，此时左右子树都已经处理完了）
}
```

---

### 2. 前序、后序与 BFS 的区别
**这是初学者最容易混淆的地方：**

*   **前序遍历 (Pre-order)**：先处理自己，再处理左边，最后处理右边。
*   **后序遍历 (Post-order)**：先处理左边，再处理右边，最后处理自己。
*   **BFS (广度优先搜索)**：是“**一层一层扫描**”。从根开始，先扫描第0层，再扫描第1层，再第2层。

❌ **纠正：你不能说“前序=BFS，后序=DFS”。**
*   前序和后序，**内部逻辑都是 DFS**（都是用递归堆栈实现的）。
*   BFS 是用**队列 (Queue)** 实现的，和递归堆栈完全不是一回事。

---

### 3. 如何理解“迭代循环”与“Index 安排”

在题目中，索引（Index）是**题目输入时固定的**。

**题目给的输入：**
```
1 1 2   (节点0: 左是1, 右是2)
2 3 4   (节点1: 左是3, 右是4)
...
```
这意味着：数据本身已经在数组里排好了。**DFS 在“循环”时，并不是自己去数 0, 1, 2, 3, 4，而是顺着数组里的“指针（索引）”往下跳。**

**从根节点0开始的 DFS 执行过程：**

1.  调用 `dfs(0)`：
    *   进入节点0，还没访问完里面的逻辑。
    *   `dfs(0)` 呼叫 `dfs(left[0])`，也就是 `dfs(1)`。
2.  进入 `dfs(1)`：
    *   `dfs(1)` 呼叫 `dfs(left[1])`，也就是 `dfs(3)`。
3.  进入 `dfs(3)`：
    *   `dfs(3)` 发现左是-1，右是-1。返回。
    *   `dfs(3)` 结束，回到 `dfs(1)`。
4.  回到 `dfs(1)`，继续执行：
    *   呼叫 `dfs(right[1])`，也就是 `dfs(4)`。
    *   `dfs(4)` 返回。
    *   `dfs(1)` 的左右都完了，执行“后序逻辑”（计算直径），返回。
5.  回到 `dfs(0)`，继续执行：
    *   呼叫 `dfs(right[0])`，也就是 `dfs(2)`。
    *   ......

---

### 4. 重点：为什么这题一定要“后序遍历”？

如果我们要算直径，**必须先知道左子树的深度和右子树的深度**。

*   **前序（先处理自己）**：你还没去求左右子树的深度，你根本算不出经过当前节点的直径，所以前序在这里没法用。
*   **后序（后处理自己）**：你递归地把左右子树都算完了，它们深度的值都已经返回给你了。此时你站在当前节点，就像站在一个十字路口，左边多长、右边多长全知道了，这时候计算“直径 = 左+右”刚好顺理成章。

### 直观总结

| 情况 | 动作时机 | 应用场景 |
| :--- | :--- | :--- |
| **前序** | 进入节点时 | 打印节点、前序创建树 |
| **后序** | 离开节点时 | **算高度、算直径、删除树（需先删子再删父）** |
| **BFS** | 按层遍历 | 找最短距离（在无权图中）、按层打印 |

**记住一句话：** 如果一个问题的答案依赖于子问题的计算结果，那就用**后序遍历（DFS）**。这就是“树形动态规划”的标准写法。
