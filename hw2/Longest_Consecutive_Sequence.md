# Longest Consecutive Sequence/最长连续序列 medium

### 📥 题意理解 (Mental Model)

本题的核心挑战在于：**如何打乱下标且不重复地找出数值上连续的最长链？**

*   **本质：** 寻找数值上构成「连续整数集合」的最大长度。
*   **关键点 1：下标不重要。** 无论原数组顺序如何，只要值能连起来（如 1, 2, 3, 4），它们就是一条序列。
*   **关键点 2：去重。** 同一个数值出现多次不增加序列长度（如 `[1, 2, 2, 3]` -> `[1, 2, 3]`），所以第一步必须去重。
*   **关键点 3：算法瓶颈。** 如果暴力遍历（对每个数都往后找），时间复杂度是 $O(n^2)$，必须通过哈希集合实现 $O(1)$ 查找，从而将时间复杂度压至 $O(n)$。

---

### 🔑 涉及知识点 (Core Concepts)

| 知识点 | 在本题中的作用 |
| :--- | :--- |
| **哈希集合 (`unordered_set`)** | 消除重复，且将“查找某个数是否存在”的性能从 `O(n)` 提升至 `O(1)`。 |
| **贪心搜索 (Greedy Search)** | 核心策略：并不是对所有数都开始计数，而是只对“序列的出生点”开始计数。 |
| **时间复杂度 Amortized O(n)** | 结论点：虽然看似有 `while` 循环，但每个数字在整个生命周期内只会被 `while` 访问一次，因此总时间复杂度为线性。 |

---

### 💻 代码细节纠正与核心逻辑 (Implementation Wisdom)

这是最容易做错或变慢的地方，注意以下三个“防火墙”：

#### 1. 核心防火墙：“如何正确找到起点”
**错误逻辑：** 对数组每个数都开始往后数（`1`数一遍，`2`数一遍...）-> `O(n^2)`。
**正确逻辑：**
```cpp
// 关键的一行代码
if (st.find(x - 1) == st.end()) { 
    // 只有当 x-1 不在集合里，说明 x 是序列的起点，才开始计数
}
```
*   **解释：** 这行代码排除了所有“中间点”。如果 `x` 的前驱 `x-1` 存在，说明 `x` 不是起点，它是属于某个已经在计数序列中的元素，跳过它能直接避免重复计数。

#### 2. 代码性能优化：预分配 (Reserve)
在 `unordered_set` 中插入大量数据时，如果触发多次哈希表重排（Rehash），性能会大打折扣。
```cpp
unordered_set<int> st;
st.reserve(nums.size()); // 提前分配空间，起飞必备！
for(int x : nums) st.insert(x);
```

#### 3. C++ 容器的选择
*   **必须用 `unordered_set`，不要用 `set`。**
    *   `unordered_set`: 底层是哈希表，查找 `O(1)`，线性 `O(n)`。
    *   `set`: 底层是红黑树，查找 `O(log n)`，整体 `O(n log n)`。
    *   虽然 `O(n log n)` 本题可能也能过，但面试时必须给出 `O(n)` 的解法。

### 💡 为什么这是 O(n)？(面试必问)

这是面试官最喜欢问的点，你可以这样回答：

1.  **集合初始化：** 遍历数组一次，插入哈希表，`O(n)`。
2.  **查找与计数：** 
    *   对于数组中的每个元素，我们**只在它是序列起点时**才会进入 `while` 循环。
    *   如果一个元素不是起点（例如 `x-1` 存在），我们直接跳过。
    *   在整个运行过程中，每个元素 `x` 实际上只被访问了 **2 次**：
        *   一次是 `st.find(x-1)` 检查（或者是 `for` 循环遍历）。
        *   一次是被其前驱（如 `x-1`）作为序列的一部分在 `while` 循环中访问。
    *   由于总的“计数访问”操作次数不会超过 `2*n`，所以摊还时间复杂度是 **`O(n)`**。

### 🎯 总结你的学习路径

1.  **如果这道题做不出来**：通常是因为试图排序（导致 `O(n log n)`）或者暴力穷举（导致 `O(n^2)`）。
2.  **如果代码写得慢**：通常是因为没用哈希表来代替数组查找。
3.  **最终复习建议**：手动在纸上模拟一遍 `[100, 4, 1, 3, 2]` 的查找过程，特别是看到 `4` 时，因为 `3` 在 `st` 里，所以 `4` 应该直接被跳过，这种“跳过逻辑”就是 $O(n)$ 实现的精髓。
