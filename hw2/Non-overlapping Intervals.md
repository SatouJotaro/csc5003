# Non-overlapping Intervals/无重叠区间
这道题目是一个经典的算法问题，通常被称为**“无重叠区间” (Non-overlapping Intervals)**。下面我为你详细解析它的含义、输入输出格式以及解题思路。

---

### 1. 题目含义
题目要求我们从给定的多个区间中，**删除最少数量的区间**，使得剩下的区间彼此之间**互不重叠**。

*   **什么叫重叠？** 如果两个区间有共同的部分（长度大于0），则视为重叠。
*   **什么叫不重叠？** 如果两个区间仅仅是在端点处接触（例如 `[1, 2]` 和 `[2, 3]`），题目明确规定这**不属于**重叠。
*   **核心目标：** 如果你保留了最多的互不重叠区间，那么你删除的区间数量就是最少的。
    *   公式：`最少删除数 = 总区间数 - 最多可保留的重叠区间数`。

---

### 2. 输入格式
*   **第一行：** 一个整数 $n$ (1 ≤ $n$ ≤ 1e5)，表示一共给出了多少个区间。
*   **接下来的 $n$ 行：** 每行包含两个整数 $start_i$ 和 $end_i$，代表第 $i$ 个区间的起始点和结束点。

---

### 3. 输出格式
*   一个整数，代表为了让剩余区间互不重叠，**必须删除的区间的最小个数**。

---

### 4. 示例解析

*   **示例 1：**
    *   输入：`[1, 2], [2, 3], [3, 4], [1, 3]`
    *   分析：`[1, 2]`, `[2, 3]`, `[3, 4]` 这三个区间互不重叠（在端点相接）。而 `[1, 3]` 会涵盖 `[1, 2]` 和部分 `[2, 3]`，导致重叠。
    *   结果：保留 `[1, 2], [2, 3], [3, 4]`，只须删掉 `[1, 3]`，答案为 **1**。

*   **示例 2：**
    *   输入：`[1, 2], [1, 2], [1, 2]`
    *   分析：三个区间完全一样，只能保留一个，必须删掉两个。
    *   结果：**2**。

*   **示例 3：**
    *   输入：`[1, 2], [2, 3]`
    *   分析：它们在 2 处相接，不属于重叠。
    *   结果：**0**。

---

### 5. 解题思路（提示）
要解决这个问题，最有效的方法是**贪心算法 (Greedy Algorithm)**：

1.  **排序：** 将所有区间按照**结束时间（end_i）进行升序排序**。
    *   *原因：* 优先选择结束时间早的区间，能为后面留下更多的空间，从而能放下更多不重叠的区间。
2.  **遍历：**
    *   维护一个变量 `end` 表示当前已选定区间的结束位置。
    *   遍历排好序的区间：
        *   如果当前区间的起始时间 >= 上一个选定区间的结束时间 `end`，说明不重叠，更新 `end` 并保留该区间。
        *   如果当前区间起始时间 < `end`，说明它与之前的区间重叠了，必须删除一个（这里选择删除结束时间较晚的那个，即当前这个，计数器加1）。

**复杂度：** 由于需要排序，总时间复杂度为 $O(n \log n)$，非常适合 $n=10^5$ 的规模。

## 知识点
这道题目是计算机科学中非常经典的**区间算法（Interval Algorithm）**问题。解决这道题涉及的核心知识点主要集中在算法设计和基础数据结构的使用。

以下是详细的知识点拆解：

### 1. 核心算法：贪心算法 (Greedy Algorithm)
这是本题的灵魂。贪心算法的思想是：**在每一步选择中，都做出在当前看来最好的选择，从而希望最终结果是全局最优的。**

*   **本题的贪心策略：** 总是选择“结束时间最早”的区间。
*   **为什么要这么做？** 如果一个区间结束得越早，它留给后续区间的“空间”就越大。这样，我们就能够尽可能多地放入后续的区间，从而达到“保留最多区间”的目标，即“删除最少区间”。
*   **与其他算法对比：**
    *   相比于暴力穷举（尝试所有删除组合），它的复杂度极低。
    *   相比于动态规划（DP），贪心算法在本题场景下逻辑更简单、运行效率更高。

### 2. 排序（Sorting）
这是实现贪心策略的**必经步骤**。没有排序，贪心算法就无法展开。

*   **排序规则：** 必须按照区间的**结束位置（end_i）进行升序排序**。
*   **核心逻辑：** 只有将区间有序化，才能保证在遍历时始终能做出正确的贪心决策。这是一个常见的算法套路：**“先排序，后遍历”**。

### 3. 数据结构基础
虽然题目看起来简单，但它涵盖了基础数据结构的组织方式：

*   **数组 (Array) / 列表 (List)：**
    *   输入存储：通常使用 `vector<pair<int, int>>` (C++) 或 `List<int[]>` (Java/Python) 来存储这些区间数据。
    *   这是最基本、空间效率最高的存储方式。
*   **结构体/对象 (Struct / Object)：** 在某些实现中，你可能会定义一个 `Interval` 类，包含 `start` 和 `end` 成员变量，这样代码可读性更高。
*   **指针或变量跟踪（状态维护）：**
    *   使用一个变量（例如 `last_end`）来记录上一个保留区间的结束位置，这是用于状态判断的简单数据结构。

### 4. 复杂度分析（计算复杂度思维）
在刷题和实际工程中，分析复杂度是必备技能：

*   **时间复杂度 ($O(n \log n)$)：** 
    *   排序过程通常使用快速排序 (Quick Sort) 或归并排序 (Merge Sort)，复杂度为 $O(n \log n)$。
    *   遍历过程为线性扫描，复杂度为 $O(n)$。
    *   总复杂度为 $O(n \log n)$，在题目 $n=10^5$ 的约束下，这是一个非常高效的解法。
*   **空间复杂度 ($O(1)$ 或 $O(n)$)：** 取决于排序算法是否原地进行。

### 5. 类比与扩展（举一反三）
通过这道题，你可以发现一系列相关的**区间问题**，它们通常都使用类似的“排序+贪心”思想：

1.  **合并区间 (Merge Intervals)：** 给定重叠的区间，将其合并成一个大区间。
2.  **会议室问题 (Meeting Rooms)：** 判断一个人能否参加所有会议，或者计算最少需要多少间会议室（这通常涉及贪心或堆/优先队列）。
3.  **箭射气球问题 (Burst Balloons)：** 逻辑与本题几乎一模一样，通过射箭的最少数量来消灭所有气球。

你在这道题中学习到的核心逻辑链是：
**问题抽象（区间重叠） $\to$ 确定贪心策略（按结束时间排序） $\to$ 排序（预处理） $\to$ 线性扫描（状态更新）**

这是初中级算法面试中最常考的模式之一，掌握了“按结束时间排序并贪心”的技巧，你就攻克了一类非常大的题型。

## 代码细节
你的直觉很敏锐，这是一个很好的尝试！但在代码实现上，确实还存在几个关键的逻辑问题，导致它无法正确运行。

我们来逐一分析并修正。

### 1. 核心问题：排序规则
默认的 `sort(intervals.begin(), intervals.end())` 在处理 `vector<vector<int>>` 时：
*   它是先比较 `intervals[i][0]`（起点），起点小的排前面。
*   只有起点相同时，才会比较 `intervals[i][1]`（终点）。

**但题目要求：必须按“结束时间”排序。** 因此，你需要自定义一个比较逻辑（Compapator）。

### 2. 逻辑问题：循环越界与状态更新
*   **越界：** 你的循环写的是 `i <= n`，这意味着会访问到 `intervals[n]`，这是越界的（合法下标是 `0` 到 `n-1`）。
*   **状态处理：** 当发生重叠时，你做了 `count++`（删掉当前的），这是对的。但是，**如果不重叠**，你需要更新 `last_end` 为当前区间的 `end`。你的逻辑部分是正确的，但要注意边界条件。

---

### 修改后的代码（C++）

```cpp
#include <iostream>
#include <algorithm>
#include <vector>

using namespace std;

int main() {
    int n;
    if (!(cin >> n) || n == 0) return 0;

    vector<vector<int>> intervals(n, vector<int>(2));
    for (int i = 0; i < n; i++) {
        cin >> intervals[i][0] >> intervals[i][1];
    }

    // 关键修正：使用 Lambda 表达式按区间结束时间（intervals[i][1]）升序排序
    sort(intervals.begin(), intervals.end(), [](const vector<int>& a, const vector<int>& b) {
        return a[1] < b[1]; // 谁结束得早，谁排在前面
    });

    int count = 0; // 记录删除了多少个
    int last_end = intervals[0][1]; // 记录上一个保留区间的结束时间

    // 从第二个区间开始遍历 (下标1到n-1)
    for (int i = 1; i < n; i++) {
        if (intervals[i][0] < last_end) {
            // 如果当前区间的开始时间小于上一个的结束时间，说明重叠了
            count++; 
            // 注意：因为我们是按结束时间排序的，这里隐含了逻辑：
            // 我们保留了结束时间早的（last_end），删除了结束时间晚的（当前这个）
        } else {
            // 不重叠，更新 last_end
            last_end = intervals[i][1];
        }
    }

    cout << count << endl;
    return 0;
}
```

### 关键点总结：

1.  **Lambda 比较器：** `[](const vector<int>& a, const vector<int>& b) { return a[1] < b[1]; }` 是 C++ 中实现自定义排序最简洁的方法。它告诉 `sort` 函数：“请只比较第二个元素”。
2.  **贪心策略的体现：**
    *   为什么重叠时不用更新 `last_end`？因为我们排过序了，`last_end` 本身就比当前区间的结束时间更早（或相等），保留它能给后续区间留出更多空间。
    *   为什么没重叠时要更新 `last_end`？因为这时候我们必须切换到当前这个区间，它是我们目前最靠右的边界。
3.  **下标保护：** 将循环控制在 `i < n` 可以有效防止程序崩溃。
