# Shortest Cycle in a Graph/图中的最短环 hard
### 一、题意理解

这道题目要求你在一个**无向图**中找到**最短环（Shortest Cycle）**的长度，即图中所有简单环中包含边数最少的那一个。如果图中不存在任何环，则返回 -1。

以下是对题目的详细解析：

### 1. 题目含义解析
*   **什么是环（Cycle）**：从一个顶点出发，经过若干条边后回到该顶点的路径。题目要求每条边最多使用一次（实际上在简单图中，找到最短环通常意味着路径上的顶点也不重复，除了起点和终点重合）。
*   **目标**：在图中所有的环中，找出边数最小的那个，并返回其长度。
*   **数据规模**：$n \le 1000, m \le 1000$。这意味着这是一个稀疏图，图的规模允许我们使用效率适中的算法。

---

### 2. 输入输出解释
*   **输入**：
    *   第一行包含两个整数 $n$ 和 $m$。$n$ 是顶点数量，$m$ 是边数量。
    *   接下来的 $m$ 行，每行两个整数 $u, v$，表示顶点 $u$ 和顶点 $v$ 之间有一条无向边。
*   **输出**：
    *   如果存在环，输出最短环的边数（一个整数）。
    *   如果不存在环（即该图是一片森林），输出 -1。

**示例分析：**
*   **Sample 1**：图中包含两个环：(0-1-2) 和 (3-4-5-6)。(0-1-2) 的长度是 3，(3-4-5-6) 的长度是 4。最短的是 3，所以输出 3。
*   **Sample 2**：只有边 (0,1) 和 (0,2)，这是一个简单的树结构（点0与1连，点0与2连），没有闭合的回路，所以输出 -1。

---

### 3. 涉及的算法与数据结构

要解决这个问题，通常有以下几种方案：

#### A. 核心算法：广度优先搜索 (BFS)
这是解决本题最常用的思路。
*   **思路**：为了找到最短环，我们可以**对每一个顶点 $i$ 作为起点执行一次 BFS**。
*   **做法**：在以 $i$ 为起点的 BFS 过程中，记录每个节点到 $i$ 的距离 `dist[]`。当我们遍历到邻接点 $v$ 时：
    1. 如果 $v$ 尚未被访问，则更新其距离并加入队列。
    2. 如果 $v$ 已经被访问过，**且 $v$ 不是 $current$ 节点的父节点**，说明我们发现了一个环。此时环的长度为：`dist[current] + dist[v] + 1`。
*   **复杂度**：对 $n$ 个点分别 BFS，单次 BFS 时间复杂度为 $O(n + m)$，总时间复杂度为 $O(n \times (n + m))$。在 $n, m \le 1000$ 的约束下，$10^6$ 次量级的运算完全可以在规定时间内运行完毕。

#### B. 数据结构
*   **邻接表 (Adjacency List)**：这是存储图的最佳方式。使用 `vector<int> adj[n]`，能够高效地遍历每个点的所有邻居。
*   **距离数组 (Distance Array)**：在 BFS 时使用 `dist[]` 数组来存储从起点到各个点的距离，并同时起到标记已访问节点的作用（初始设为 -1 或无穷大）。
*   **队列 (Queue)**：BFS 的标准数据结构，用于按层遍历图。

#### C. 注意点 (特殊算法提示)
*   **Floyd-Warshall 变体（Floyd算法）**：
    通常 Floyd 用于求全源最短路径，复杂度为 $O(n^3)$。如果数据范围再小一点（如 $n \le 400$），可以使用 Floyd 求最短环：
    在更新距离的过程中，如果发现 $dist[i][j] + weight(j, k) + weight(k, i) < min\_cycle$，即可更新最短环。但对于本题，BFS 更优且更符合稀疏图的特性。

### 总结建议
针对该题目，推荐采用 **BFS 循环执行法**：即以图中的每一个点作为 BFS 起点，通过 BFS 寻找能形成的最小环长度，最后取所有结果的最小值。

### 三、代码细节纠正

#### 核心逻辑纠正 —— 关于 `visited` 和 `dist`
你原来的代码中，`visited` 数组在整个程序中只初始化一次。这意味着如果你发现了一个环，后续的节点如果被包含在之前的 BFS 路径中，就不会作为新的起点去搜索，这可能漏掉其他更短的环。

**修改方案**：
1.  **移除外层的 `visited` 数组限制**：我们必须对“每一个节点”都进行 BFS。
2.  **重置辅助数组**：每次 BFS 循环时，必须重置 `dist`（或 `depth`）数组，通常初始化为 -1 表示未访问。

#### 修正后的代码：

```cpp
#include <iostream>
#include <vector>
#include <queue>
#include <algorithm> // 为了使用 min

using namespace std;

const int INF = 1e9;

int main() {
    int n, m;
    if (!(cin >> n >> m)) return 0;
    
    vector<vector<int>> adj(n);
    for (int i = 0; i < m; i++) {
        int u, v;
        cin >> u >> v;
        adj[u].push_back(v);
        adj[v].push_back(u);
    }
    
    int min_cycle = INF;
    
    // 【修改点】：对每一个点作为起点进行 BFS
    // 这样能保证我们计算出经过所有情况的最短环
    vector<int> dist(n);
    vector<int> parent(n);
    for (int i = 0; i < n; i++) {
        fill(dist.begin(), dist.end(), -1);
        fill(parent.begin(), parent.end(), -1);
        
        queue<int> q;
        
        dist[i] = 0;
        q.push(i);
        
        while (!q.empty()) {
            int u = q.front();
            q.pop();
            
            for (int v : adj[u]) {
                if (dist[v] == -1) {
                    // 情况1：v 未被访问，正常入队
                    dist[v] = dist[u] + 1;
                    parent[v] = u;
                    q.push(v);
                } else if (v != parent[u]) {
                    // 情况2：v 已访问且不是父节点，发现环
                    // 环长 = 到u的距离 + 到v的距离 + 1
                    min_cycle = min(min_cycle, dist[u] + dist[v] + 1);
                }
            }
        }
    }
    
    if (min_cycle == INF) cout << -1 << endl;
    else cout << min_cycle << endl;
    
    return 0;
}
```

### 为什么这样做复杂度没问题？
*   **复杂度分析**：外层循环 $N$ 次，内层 BFS 遍历边 $O(M)$ 和点 $O(N)$。总时间复杂度为 $O(N(N+M))$。
*   对于 $N=1000$， $1000 \times 2000 = 2 \times 10^6$ 次运算，完全可以在 1 秒内通过（通常限制为 $10^7 \sim 10^8$ 量级）。
*   这种方法虽然看起来有些“暴力”，但它是处理无权图最短环最标准且鲁棒性最强的方法。

### BFS细节
没关系，这些概念在 BFS（广度优先搜索）中确实比较容易混淆。我们把它们拆开来细说。

### 1. `dist` 数组到底存了什么？
`dist` 数组的作用有两个，它是图搜索算法中非常经典的设计：
*   **判断是否“待处理”**：初始值全是 `-1`。当 `dist[v] == -1` 时，代表“这个点还没被发现过”（也就是还没进过队列）。
*   **记录距离**：一旦被发现，`dist[v]` 就记录了**从起点 `start` 到达这个点 `v` 最少需要经过多少条边**。
*   **为什么 `dist[v] == -1` 成立？** 因为 BFS 是一层一层往外扩散的。如果一个点 `v` 还没被访问过，那么它一定还没有被计算过到起点的距离，所以初始化为 `-1` 是最自然的标记。

### 2. 为什么要用 `parent` 数组？
在无向图中，边是双向的。假设有边 `(u, v)`：
*   你从 `u` 走到了 `v`，那么 `u` 是 `v` 的“父节点”。
*   但是 `v` 的邻居列表里一定也包含 `u`。
*   当你处理完 `u`，开始处理 `v` 的时候，BFS 会看到 `v` 的邻居里有一个 `u`。
*   **关键点**：如果不加 `parent` 判断，算法会错误地认为从 `v` 又走回 `u` 形成了一个环（长度为 2）。但实际上这只是那条边原路返回，并不是题目要求的“环”。
*   因此，`parent[v] = u` 的意义是：**“记录我是从哪个点跳过来的”**。如果我再次看见父节点，直接忽略，避免走回头路。

---

### 3. 给你的“场景模拟” (以起点 `i` 为例)

想象你在一个迷宫里，手拿一根绳子（BFS 队列）：

1.  **起点**：你站在 `i` 点，`dist[i] = 0`，绳子系在 `i` 上。
2.  **扩散**：你向四周看，发现邻居 `A`。因为 `dist[A]` 是 `-1`，说明 `A` 是个新房间。你记下 `dist[A] = dist[i] + 1`（也就是 1），并记下“我是从 `i` 走到 `A` 的”，即 `parent[A] = i`。
3.  **继续**：后来你从 `A` 走到了 `B`，`dist[B] = 2`，`parent[B] = A`。
4.  **发现环**：
    *   此时你站在 `B` 点，发现邻居里有一个 `C`。
    *   你一看 `dist[C]` 不是 `-1`（说明你在之前走别的路时已经到过 `C` 了）。
    *   你也检查了 `C` 是不是 `parent[B]` （即 `C` 是不是 `A`）。
    *   **如果不是父节点**：这意味着你通过另一条路也到达了 `C`，**闭环产生了！**
    *   此时，`dist[B]` 是你从起点到 `B` 的步数，`dist[C]` 是你之前从起点到 `C` 的步数，加上中间连接 `B` 和 `C` 的那条边（计为 1），总共就是 `dist[B] + dist[C] + 1`。

---

### 4. 为什么判断条件是 `dist[u] + dist[v] + 1`？

这是一个非常优雅的数学结论。

*   **起点 $S$ 到 $u$ 的距离**：$dist[u]$
*   **起点 $S$ 到 $v$ 的距离**：$dist[v]$
*   **连接 $u, v$ 的那条边**：长度为 $1$

当你已经在 BFS 的过程中发现了一个闭环（即 $u$ 和 $v$ 互相连接，且它们都在以 $S$ 为起点的搜索树中），这个环的长度就是：**“起点到 $u$ 的路径” + “起点到 $v$ 的路径” + “$u$ 和 $v$ 之间的边”**。

### 总结图解
```text
      S (起点)
     / \
    ... ...
    /     \
   u-------v
```
*   `u` 和 `v` 都是通过不同的分支从 `S` 走过来的。
*   如果这时你正站在 `u`，发现邻居 `v` 已经被访问过了，且 `v` 不是从你这里过来的（不是父节点），那么你立刻就找到了一个环。
*   这个环的长度就是以 $u, v$ 为终点的这两条路合起来的长度。

**一句话总结：**
`dist` 存的是**找路的进度**（兼做访问标记），`parent` 存的是**来时的路**（为了防止原路返回）。这套机制就像你在走迷宫时，在地上画线做标记，防止自己原地转圈，并利用标记计算路径长度。
