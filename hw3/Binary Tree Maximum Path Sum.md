# Binary Tree Maximum Path Sum / 二叉树中的最大路径和 (Hard)

## 一、 题意理解
1. **什么是“路径”？**
   - 路径是树中任意两个节点之间联通的**简单路径**。
   - 限定条件：每个节点最多出现一次（不能走回头路）。
   - 自由度：路径**不一定**要经过根节点，也**不一定**要走到叶子节点，它可以开始于任意节点，结束于任意节点。
2. **目标要求：**
   - 找到一条路径，使得路径上所有节点的值相加之和最大。
3. **输入输出格式说明（ACM/本地评测机模式）：**
   - 本题的笔记基于数组实现树（而非 LeetCode 上的指针形式）。
   - 节点关系由三个整数构成：`val_i left_i right_i`（当前值，左孩子索引，右孩子索引）。空节点用 `-1` 表示。

---

## 二、 涉及知识点
本题是经典的**树形 DP（动态规划）**结合**深度优先搜索（DFS - 后序遍历）**的问题。核心在于区分清楚“路径的拐弯点”和“对父节点的贡献”。

1. **核心概念剖析（必须彻底分清的两笔账）：**
   对于任意一个节点 `node`，我们在遍历到它时，要计算两个不同的值：
   - **① 以该节点为“最高点（拐弯处）”的局部最大路径和：**
     - 形状类似 `/\`（左子树 -> 当前节点 -> 右子树）。
     - 计算公式：`当前节点值 + 左子树的正向贡献 + 右子树的正向贡献`。
     - **作用：** 用这个值去**挑战并更新全局最大值 `max_sum`**。
   - **② 该节点向上（对它的父节点）能提供的最大贡献值（Gain）：**
     - 形状类似 `/` 或 `\`（包含当前节点，且最多只包含左子树或者右子树中的一条边）。
     - 计算公式：`当前节点值 + max(左子树正向贡献, 右子树正向贡献)`。
     - **作用：** 作为递归函数的 **`return` 返回值**，供父节点计算使用。*（注意：因为路径不能分叉，所以向上返回时绝对不能同时加上左右子树的贡献！）*

2. **贪心思想 / 剪枝（忽略负增益）：**
   - 如果某个子树算出来的最大增益是**负数**，那么把它连入当前路径只会导致总和越加越小。
   - 因此，遇到负增益直接舍弃，将其视为 `0`（即不走那个子节点）：`max(0, maxGain(child))`。

3. **复杂度：**
   - 时间复杂度：$O(N)$，每个节点在 DFS 中只被访问一次计算贡献。
   - 空间复杂度：$O(H)$，$H$ 为树的高度，主要是递归系统调用的栈空间（最坏链表情况退化为 $O(N)$）。

## 三、 代码细节纠正（防坑指南）
在实现和复盘代码时，需要特别注意以下几个细节和易错点：

### 1. 全局最大值的初始化坑点
```cpp
int max_sum = INT_MIN; // 绝对不能初始化为 0！
```
- **纠正/警惕：** 树中的节点值可能是全负数（例如示例中全都是 -10, -5, -3）。如果 `max_sum` 初始化为 `0`，那么对于全负数的树，最后输出的将会是 `0`（而不是最大的那一个负数），这就大错特错了。一定要用 `INT_MIN`。

### 2. 递归 Base Case 处理
```cpp
if (index == -1) return 0; // 遇到空节点（对应 LeetCode的 root == nullptr）
```
- **细节：** 空节点对路径的贡献值为 0，作为递归的终点，防止数组越界。

### 3. C++ 传参性能优化细节
```cpp
int maxGain(const vector<TreeNode>& nodes, int index) { ... }
const TreeNode& node = nodes[index];
```
- **纠正/优化：** 
  - 传递 `nodes` 数组时必须用**引用传参 (`const vector...&`)**，否则每次递归都会发生数组深拷贝，导致严重超时（TLE / MLE）。
  - 取出当前节点时使用 `const TreeNode&` 可以避免创建一个临时的结构体对象，进一步优化性能。

### 4. 主函数中 vector 的内存预分配
```cpp
nodes.reserve(n);
for(...) { ... nodes.emplace_back(val, left, right); }
```
- **细节：** 使用了 `reserve(n)` 和 `emplace_back`。因为提前知道有 `n` 个节点，`reserve` 可以防止 `vector` 在 `push_back` 的过程中发生极其耗时的动态扩容（重新分配内存 + 拷贝旧数据）。`emplace_back` 则省去了拷贝构造，是 C++ 中的优秀编码习惯。

### 5. 一张图总结代码执行逻辑（脑内速记）：
```text
进入 maxGain(node):
  1. 如果 node 越界 -> 返回 0
  2. 左边能给我拿多少？ left_gain = max(0, maxGain(左子))
  3. 右边能给我拿多少？ right_gain = max(0, maxGain(右子))
  4. 顺手更新历史最高纪录： max_sum = max(max_sum, node.val + left_gain + right_gain)
  5. 告诉老爸我能贡献多少： return node.val + max(left_gain, right_gain)
```
