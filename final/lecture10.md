# 贪心（Greedy） 和 分治（Divide-and-Conquer）

| 范式                  | 子问题结构           | 特点        |
| ------------------- | --------------- | --------- |
| Divide-and-Conquer  | 独立子问题，树状分支（无重叠） | 子问题互不共享   |
| Dynamic Programming | 子问题有重叠，DAG 结构   | 共享子问题，记忆化 |
| Greedy              | 线性链式，只有一个子问题    | 每步做最优局部决策 |


## 1. Candy（糖果分配）
问题： 给 n 个孩子分糖果，每人至少 1 颗，评分高于相邻孩子的需获得更多糖果，求最小总糖果数 。

解法（经典双向贪心模式）：

Step 1（左→右）： 计算 left[i]，满足左侧约束：若 ratings[i] > ratings[i-1]，则 left[i] = left[i-1] + 1，否则为 1

Step 2（右→左）： 计算 right[i]，满足右侧约束：若 ratings[i] > ratings[i+1]，则 right[i] = right[i+1] + 1，否则为 1

Step 3（合并）： candy[i]=max(left[i], right[i])

核心思想： 约束来自两个方向时，分别独立求解，再取每位置的最大值。

## 2. Strange Test（奇怪测试）
问题： 给定 a<b，每次可以将 a+1、b+1 或 a = a | b（按位OR），求使 a=b 的最少操作数 

解法：
不用 OR 操作时，答案为 b−a

第三个操作最多用一次（因为用完后 a≥b，之后只能用第二操作）

若用 OR，枚举 a' 从 a 到 b，对每个 a' 贪心构造使 a' | b' 最小的 b'（从高位到低位，遇到 a' 某位为 1 而 b 某位为 0 时，强制 b' 该位为 1 并停止）

最终取所有方案的最小值

## 3. Score Maximization（分数最大化）
问题： 数组 a，二进制数组 b，至多 k 次操作（每次将 bi =1 的位置 +1），最大化 maxi(ai+median(ci))

解法：
排序后，最优分数一定是 an+median(cn)（最大元素 + 剩余元素的中位数）

分两种情况取最大：

Case 1： 把所有 k 次操作全加在某个 bi=1 的元素上，使其成为最大值，候选值 = ai+k+median(ci)

Case 2： 保持 an 为最大，用 k 次操作提升中位数，用二分搜索找可达的最大中位数 x

中位数可达性判断（二分 check 函数）：

目标：让 cn 中至少 ⌊(n+1)/2⌋ 个元素 ≥x
对 <x 且 bi=1 的元素，优先选值最大的（因为代价 x−ai 最小），贪心消耗预算 k

## 4. Count of Smaller Numbers After Self
问题： 对每个 index i，统计其右边有多少个元素比 nums[i] 小 。

解法（归并排序计数）：

每个元素携带原始下标 (value, index)

归并两个有序半段时，左半 = 原位置靠前的元素，右半 = 原位置靠后的元素

当左半元素 left[i] 被放入时，已经被取出的右半元素数量，就是 left[i] 右边比它小的数的个数

累加到 ans[left[i].index]

## 5. Count of Range Sum
问题： 给定数组和范围 [lower, upper]，统计区间和在范围内的对数 。

解法：
转化为前缀和问题：统计 i<j 且 lower≤pre[j]−pre[i]≤upper 的对数

对前缀和数组做归并排序，归并时左半 = 较小 index 的前缀和，右半 = 较大 index 的前缀和

对每个左半元素 pre[i]，在右半（已排序）中用双指针找满足 pre[i]+lower≤pre[j]≤pre[i]+upper 的 j 数量，即 j2−j1


---
贪心：认清约束方向（单向 vs 双向），双向约束必须分开处理后取 max

分治：归并排序不只能排序，还能在 merge 阶段统计跨半段的信息（逆序数、区间和对数）

二分搜索 + 贪心：常见组合，二分答案，贪心验证可行性

时间复杂度：归并排序类题目均为 O(nlogn)
