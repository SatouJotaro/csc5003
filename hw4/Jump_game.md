# LeetCode 55 跳跃游戏 详解

## 📋 题目理解

**输入：** 一个非负整数数组 `nums`  
**输出：** `bool` 值，能到达最后下标返回 `true`，否则 `false`

**关键理解：**
- `nums[i]` 表示从位置 `i` **最多**能跳多远（不是必须跳这么远！）
- 可以跳 `1` 步到 `nums[i]` 步之间的**任意距离**

---

## ❌ 你的代码问题分析

```cpp
while(i < n){
    result += nums[i];   // ❌ 累加跳跃值，没有意义
    i += nums[i];        // ❌ 每次强制跳最大距离
}                        // ❌ 没有return语句
                         // ❌ nums[i]=0时死循环！
```

**根本错误：** 你每次都跳最大距离，但实际上跳少一点可能绕过中间的"陷阱0"到达终点

---

## 💡 正确思路 —— 贪心算法

**核心思想：** 维护一个变量 `maxReach`，表示**当前能到达的最远下标**

```
遍历每个位置 i：
  1. 如果 i > maxReach → 说明无法到达位置i → 返回false
  2. 更新 maxReach = max(maxReach, i + nums[i])
  3. 如果 maxReach >= n-1 → 能到达终点 → 返回true
```

### 图解示例

**示例1：** `[2, 3, 1, 1, 4]`
```
i=0: maxReach = max(0, 0+2) = 2
i=1: 1<=2可达, maxReach = max(2, 1+3) = 4 ≥ 4 → true ✅
```

**示例2：** `[3, 2, 1, 0, 4]`
```
i=0: maxReach = max(0, 0+3) = 3
i=1: 1<=3可达, maxReach = max(3, 1+2) = 3
i=2: 2<=3可达, maxReach = max(3, 2+1) = 3
i=3: 3<=3可达, maxReach = max(3, 3+0) = 3  ← 被0困住了
i=4: 4 > 3 无法到达 → false ❌
```

---

## ✅ 正确C++代码

```cpp
class Solution {
public:
    bool canJump(vector<int>& nums) {
        int n = nums.size();
        int maxReach = 0;  // 当前能到达的最远下标
        
        for (int i = 0; i < n; i++) {
            // 当前位置已经超出能到达的范围
            if (i > maxReach) return false;
            
            // 更新最远可达位置
            maxReach = max(maxReach, i + nums[i]);
            
            // 已经能覆盖到终点
            if (maxReach >= n - 1) return true;
        }
        
        return true;
    }
};
```

---

## 📊 复杂度

| 复杂度 | 值 |
|--------|-----|
| 时间复杂度 | O(n) — 只遍历一次 |
| 空间复杂度 | O(1) — 只用了一个变量 |

---

## 🎯 知识点总结

**贪心算法** — 局部最优推导全局最优  
> 每一步都尽量扩大"能到达的最远范围"，若某个位置无法被覆盖则必然不可达

# LeetCode 45 跳跃游戏 II 详解

## 📋 题目理解

**输入：** 非负整数数组 `nums`  
**输出：** 到达最后下标的**最小跳跃次数**（保证一定能到达）

**与上题区别：**
| | 跳跃游戏I | 跳跃游戏II |
|--|--|--|
| 问题 | 能否到达 | 最少几步到达 |
| 返回值 | bool | int |

---

## 💡 解题思路 —— 贪心算法

**核心思想：** 把跳跃过程理解为"区间扩展"

```
把数组划分成若干跳跃区间：

[2, 3, 1, 1, 4]
 ↑
第0跳：只在 index=0，curEnd=0

第1跳能到达：index 1~2（因为nums[0]=2）
              curEnd=2

第2跳能到达：index 3~4（从1~2区间内能到最远4）
              curEnd=4 ≥ n-1，结束！
```

**维护三个变量：**
```
jumps   = 已跳次数
curEnd  = 当前这一跳能到达的最远边界
maxReach = 下一跳能到达的最远位置
```

**关键规则：**
> 当 `i == curEnd` 时，说明当前这一跳的范围走完了，**必须再跳一次**，把 `curEnd` 更新为 `maxReach`

---

## 📊 图解过程

**示例1：** `[2, 3, 1, 1, 4]`，n=5，终点=4

```
初始: jumps=0, curEnd=0, maxReach=0

i=0: maxReach = max(0, 0+2) = 2
     i==curEnd(0)? 是 → jumps=1, curEnd=2
     ┌─────────────────────────────┐
     │ 第1跳覆盖范围: [0 → 2]      │
     └─────────────────────────────┘

i=1: maxReach = max(2, 1+3) = 4
     i==curEnd(2)? 否

i=2: maxReach = max(4, 2+1) = 4  
     i==curEnd(2)? 是 → jumps=2, curEnd=4 ≥ 4 结束!
     ┌─────────────────────────────┐
     │ 第2跳覆盖范围: [3 → 4] ✅   │
     └─────────────────────────────┘

输出: 2
```

**示例2：** `[2, 3, 0, 1, 4]`

```
i=0: maxReach=2, i==curEnd → jumps=1, curEnd=2
i=1: maxReach = max(2,1+3) = 4
i=2: maxReach = max(4,2+0) = 4
     i==curEnd(2) → jumps=2, curEnd=4 ≥ 4 结束!

输出: 2
```

---

## ✅ C++ 代码

```cpp
class Solution {
public:
    int jump(vector<int>& nums) {
        int n = nums.size();
        int jumps = 0;       // 跳跃次数
        int curEnd = 0;      // 当前跳能到达的边界
        int maxReach = 0;    // 下一跳能到达的最远处
        
        // 注意：只遍历到 n-2，不需要在终点再跳
        for (int i = 0; i < n - 1; i++) {
            // 更新下一跳最远能到哪
            maxReach = max(maxReach, i + nums[i]);
            
            // 走到当前跳的边界了，必须再跳一次
            if (i == curEnd) {
                jumps++;
                curEnd = maxReach;
                
                if (curEnd >= n - 1) break;  // 已能覆盖终点，提前结束
            }
        }
        
        return jumps;
    }
};
```

---

## ❓ 几个容易疑惑的点

**Q1：为什么循环到 `n-2` 而不是 `n-1`？**
```
我们已经在终点了，不需要再考虑从终点跳出去
循环到 n-2 就够了
```

**Q2：为什么贪心是正确的？**
```
每次到达边界时，我们选择"下一跳尽可能跳远"
跳得更远只会更好，不可能更差
→ 局部最优 = 全局最优
```

**Q3：和上一题的关系？**
```
跳跃游戏I：maxReach能否覆盖终点（能/不能）
跳跃游戏II：需要扩展几次区间才能覆盖终点（计数）
```
