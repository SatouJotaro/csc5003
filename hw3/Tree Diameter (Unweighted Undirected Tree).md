这道题目要求你计算给定**树（Tree）的直径**。下面我为你详细拆解题目的含义以及输入输出规则：

### 1. 题目背景与含义
*   **什么是树（Tree）**：题目中明确指出这是一个“无向、连通、无环”的图。这意味着图中任意两个节点之间都有且仅有一条唯一的路径。
*   **什么是直径（Diameter）**：树中任意两个节点之间的距离（即路径上边的条数）的最大值。简单来说，就是这棵树里“最远的一对点”之间，一共经过了多少条边。
*   **任务**：给定这棵树的结构（点数和连接方式），你需要找出这个最大距离。

---

### 2. 输入输出说明

#### **输入格式 (Input Format)**
*   **第一行**：一个整数 $n$，代表树中有 $n$ 个节点（编号为 $1$ 到 $n$）。
    *   范围限制：$1 \le n \le 2 \times 10^5$。
*   **接下来的 $n-1$ 行**：每一行包含两个整数 $u$ 和 $v$，表示节点 $u$ 和节点 $v$ 之间有一条无向边相连。
    *   因为是树，所以 $n$ 个节点一定会有 $n-1$ 条边。

#### **输出格式 (Output Format)**
*   **输出**：一个整数，即该树的直径（即最长路径包含的边数）。

---

### 3. 样例图解 (Sample #1)
**输入：**
```
5
1 2
2 3
2 4
4 5
```
**分析：**
*   这棵树的结构如下：
    *   1 - 2
    *   2 - 3
    *   2 - 4
    *   4 - 5
*   我们可以列出一些路径长度：
    *   1 到 3：长度 2 (1-2, 2-3)
    *   1 到 5：长度 3 (1-2, 2-4, 4-5)
    *   3 到 5：长度 3 (3-2, 2-4, 4-5)
*   **直径**：树中最长的路径是 3-2-4-5 或 1-2-4-5，长度均为 **3**，所以输出 3。

---

### 4. 解题思路提示 (进阶)
对于 $n$ 最大达到 $2 \times 10^5$ 的规模，暴力枚举每一对点是不现实的（因为有 $O(n^2)$ 对点）。常用的高效解法通常有两种：

1.  **两次 BFS 或 DFS**：
    *   从任意一点 $x$ 出发，找到距离它最远的点 $y$。
    *   再从 $y$ 出发，找到距离 $y$ 最远的点 $z$。
    *   那么 $y$ 与 $z$ 之间的距离就是树的直径。
    *   复杂度：$O(n)$。

2.  **动态规划 (树形DP)**：
    *   对于每个节点，记录其向下延伸的最长路径和次长路径，相加即为经过该点的最长路径。
    *   复杂度：$O(n)$。

这两种方法都能在规定时间内高效求解。

## hw2.2 vs hw2.1
虽然这两道题的核心目标都是计算“树的直径”，但它们在**数据结构定义**和**题目表现形式**上有显著区别，这会影响你编写代码时的逻辑。

以下是它们的主要区别对比：

### 1. 结构化定义的不同
*   **第一道题（无根树/一般图结构）**：
    *   **定义**：题目给的是一个无向、连通的图（树）。
    *   **没有“父子”之分**：题目只给了边 `u v`，它们是平等的，你需要自己通过搜索（BFS/DFS）去建立关系。
    *   **结构不固定**：节点可以有任意多的分支（一个节点可以连多个邻居）。
*   **第二道题（二叉树/特定父子结构）**：
    *   **定义**：题目已经明确给出了“二叉树”的父子层级关系。
    *   **有“父子”之分**：每个节点明确指出它的左孩子是谁、右孩子是谁（或为 -1 表示没有）。
    *   **结构受限**：每个节点最多只有 2 个孩子。

### 2. 输入方式差异（最直接的区别）
*   **第一道题的输入是“边列表”（Edge List）**：
    *   告诉你哪些点连在一起，你需要用**邻接表 (Adjacency List)** 来存储图（例如 `vector<int> adj[N]`）。
*   **第二道题的输入是“节点定义”（Node Definition）**：
    *   直接给出节点编号及其对应的孩子编号。这意味着这本质上是一个**数组表示的树**，你可以直接通过 `tree[i].left` 或 `tree[i].right` 来访问，不需要自己建图。

### 3. 解题思路的区别

*   **对于第一道题（无向大树）**：
    *   **核心逻辑**：由于树没有“根”，通常使用**两次 BFS/DFS**：
        1. 随意选一点 $u$，找离它最远的点 $v$。
        2. 再从 $v$ 出发找到离它最远的点 $w$。
        3. $dist(v, w)$ 即为直径。

*   **对于第二道题（二叉树）**：
    *   **核心逻辑**：通常使用**递归法（后序遍历）**：
        *   定义一个函数 `depth(root)` 返回以该节点为根的子树的高度。
        *   对于任何节点 `node`，它跨越的路径长度为 `depth(node.left) + depth(node.right)`。
        *   在递归的同时，更新全局的最大直径变量 `max_diameter = max(max_diameter, left_height + right_height)`。

### 总结比较表

| 特性 | 第一题 (Tree Diameter) | 第二题 (Binary Tree Diameter) |
| :--- | :--- | :--- |
| **树的类型** | 一般的无向无环图 | 二叉树（有左右孩子之分） |
| **输入方式** | 给出所有连通的边 | 给出每个节点的左右子节点 |
| **存储方式** | 建议用邻接表 (`vector<int> adj[]`) | 用数组或结构体 `struct Node { int l, r; }` |
| **典型解法** | 两次 BFS 或 DFS | 递归后序遍历 (计算高度并尝试更新答案) |

**一句话总结：**
第一题是 **“路网”** 式的输入，需要自己理清谁和谁连着；第二题是 **“家谱”** 式的输入，直接告诉你谁是谁的下一代，递归处理起来会更直观。

## 两种解法
针对你关心的第一题（无向树直径），这两种方法都是 $O(n)$ 的，且在面试和竞赛中都非常重要。以下是具体的 C++ 实现。

### 方法一：两次 BFS/DFS（更直观）
这种方法不需要关心树的形态，只需要找到距离最远的点。

```cpp
#include <iostream>
#include <vector>
#include <algorithm>
#include <queue>

using namespace std;

const int MAXN = 200005;
vector<int> adj[MAXN];
int dist[MAXN];

// 这里使用 BFS，如果用 DFS 除了递归写法外，逻辑完全一致
pair<int, int> bfs(int start, int n) {
    fill(dist, dist + n + 1, -1);
    queue<int> q;
    
    q.push(start);
    dist[start] = 0;
    
    int farthestNode = start;
    while (!q.empty()) {
        int u = q.front(); q.pop();
        
        if (dist[u] > dist[farthestNode]) {
            farthestNode = u;
        }
        
        for (int v : adj[u]) {
            if (dist[v] == -1) {
                dist[v] = dist[u] + 1;
                q.push(v);
            }
        }
    }
    return {farthestNode, dist[farthestNode]};
}

int main() {
    int n; cin >> n;
    for (int i = 0; i < n - 1; ++i) {
        int u, v; cin >> u >> v;
        adj[u].push_back(v);
        adj[v].push_back(u);
    }
    
    // 第一次：从任意点（比如点1）出发找最远点y
    pair<int, int> p1 = bfs(1, n);
    // 第二次：从y出发找最远点z，距离即直径
    pair<int, int> p2 = bfs(p1.first, n);
    
    cout << p2.second << endl;
    return 0;
}
```

### 方法二：树形 DP（更强大）
树形 DP 的精髓在于：**经过节点 `u` 的最长路径 = 以子节点为起点的向下最长路径 + 次长路径**。这种方法在处理带权边或路径计数问题时更有扩展性。

```cpp
#include <iostream>
#include <vector>
#include <algorithm>

using namespace std;

const int MAXN = 200005;
vector<int> adj[MAXN];
int ans = 0;

// dfs 返回以 u 为顶点的子树中，从 u 出发向下的最长链长度
int dfs(int u, int p) {
    int max1 = 0; // 最长向下路径
    int max2 = 0; // 次长向下路径
    
    for (int v : adj[u]) {
        if (v == p) continue; // 防止回溯到父节点
        
        int d = dfs(v, u) + 1;
        
        if (d > max1) {
            max2 = max1;
            max1 = d;
        } else if (d > max2) {
            max2 = d;
        }
    }
    
    // 该点的最长路 = 最长分支 + 次长分支
    ans = max(ans, max1 + max2);
    
    return max1; // 返回给上一层调用者，只能带走最长的那条分支
}

int main() {
    int n; cin >> n;
    // 如果是题目要求的“输入边”，处理逻辑如下：
    for (int i = 0; i < n - 1; ++i) {
        int u, v; cin >> u >> v;
        adj[u].push_back(v);
        adj[v].push_back(u);
    }
    
    dfs(1, -1); // 从根（假设为1）开始遍历
    cout << ans << endl;
    return 0;
}
```

### 两种方法的选择建议：

1.  **两次 BFS/DFS**：
    *   **优点**：逻辑极其简单，不需要维护额外的状态（比如处理次长路径），不容易写错。
    *   **限制**：要求图必须是无权图（或所有边权均为正）。

2.  **树形 DP**：
    *   **优点**：在解决“加权路径”或者“计算路径数量”等更复杂问题时，它是标准解法。它只需要遍历一次，且能顺便算出每个子树的信息。
    *   **注意**：代码中一定要处理好 `max1` 和 `max2` 的更新逻辑。

**如果你只是单纯求直径，建议背诵“两次 BFS/DFS”的写法；如果你在学习 DP 相关知识，一定要掌握“树形 DP”的这种更新逻辑。**
