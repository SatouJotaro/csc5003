这道题是经典的“最长回文子串”问题（Longest Palindromic Substring）。下面我为你从题目理解、知识背景、解题思路到代码实现进行详细拆解。

---

### 1. 题目理解
*   **输入**：一个仅包含小写英文字母的字符串 $S$（长度 $n$ 不超过 1000）。
*   **输出**：一个整数，代表 $S$ 中最长的回文子串的长度。
    *   *什么是回文子串？* 正着读和反着读都一样的子串，比如 `aba` 是回文，`abacaba` 也是回文。

---

### 2. 核心知识点
*   **双指针法 / 中心扩展法 (Expand Around Center)**：这是解决回文问题最直观、最高效的方法之一（时间复杂度 $O(n^2)$）。
*   **回文的特性**：回文中心只有两种形式：
    1.  **奇数长度回文**：中心是一个字符（例如 `aba`，中心是 `b`）。
    2.  **偶数长度回文**：中心是两个字符之间的空隙（例如 `abba`，中心是 `bb` 之间）。

---

### 3. 解题思路（中心扩展法）
由于字符串长度 $n \le 1000$， $O(n^2)$ 的复杂度约为 $10^6$，在 1 秒的时限内绰绰有余。

**具体逻辑：**
1. 遍历字符串中的每一个位置 $i$（作为潜在的中心点）。
2. 对于每个 $i$：
    *   **以 $i$ 为中心**（奇数情况）：向左右两边同时扩展，直到字符不相等或越界。
    *   **以 $i$ 和 $i+1$ 中间为中心**（偶数情况）：向左右两边同时扩展，直到字符不相等或越界。
3. 在扩展过程中记录下每一次找到的回文长度，并维护一个 `max_len` 变量保存最大值。
4. 最后输出 `max_len`。

---

### 4. C++ 代码实现

```cpp
#include <iostream>
#include <string>
#include <vector>
#include <algorithm>

using namespace std;

// 辅助函数：从中心向外扩展，返回回文长度
int expandAroundCenter(const string& s, int left, int right) {
    while (left >= 0 && right < s.length() && s[left] == s[right]) {
        left--;
        right++;
    }
    // 返回长度：扩展停止时，left 和 right 都多走了一步，
    // 因此总长度是 (right - 1) - (left + 1) + 1 = right - left - 1
    return right - left - 1;
}

int main() {
    string s;
    cin >> s;
    int n = s.length();
    if (n == 0) {
        cout << 0 << endl;
        return 0;
    }

    int max_len = 0;
    for (int i = 0; i < n; i++) {
        // 情况1：回文长度为奇数 (中心是一个字符)
        int len1 = expandAroundCenter(s, i, i);
        // 情况2：回文长度为偶数 (中心是两个字符之间)
        int len2 = expandAroundCenter(s, i, i + 1);
        
        max_len = max({max_len, len1, len2});
    }

    cout << max_len << endl;
    return 0;
}
```

---

### 5. 代码要点解析
*   **`expandAroundCenter` 函数**：
    *   输入 `left` 和 `right` 指针。如果是奇数长度，传入 `(i, i)`；如果是偶数，传入 `(i, i+1)`。
    *   `while` 循环确保左右字符相等且不越界。
    *   返回长度的公式 `right - left - 1` 是因为当 `s[left]` 不等于 `s[right]` 时，实际的回文范围是 `[left+1, right-1]`。
*   **时间复杂度**：外层循环执行 $n$ 次，每次内部扩展最多执行 $n$ 次，总复杂度为 $O(n^2)$。
*   **空间复杂度**：只用了几个变量，空间复杂度为 $O(1)$。

### 总结
这道题考查的是对字符串边界的处理能力。中心扩展法之所以好用，是因为它避开了复杂的预处理，直接通过分两种情况讨论中心点，完美覆盖了所有回文的可能性。如果题目中的 $n$ 达到 $10^5$ 级别，则需要使用更高级的 **Manacher 算法**，但对于这道题，$O(n^2)$ 是最稳妥也是最推荐的做法。


在 $n \le 1000$ 时，$O(n^2)$ 的算法可以完美运行。但当约束条件变为 **$n \le 100,000$** 时，$O(n^2)$（即 $10^{10}$ 次运算）会直接超时（TLE）。

此时，我们需要使用专门求解最长回文子串的线性时间复杂度算法：**Manacher 算法（马拉车算法）**。它的时间复杂度为 **$O(n)$**。

---

### 1. 核心瓶颈与解决思路
*   **为什么快不起来？** $O(n^2)$ 的算法做了很多重复对比。例如在 `abacaba` 中，当我们计算中心点 `c` 附近的回文时，我们已经知道它是回文了，没必要从头再扫一遍。
*   **Manacher 算法的聪明之处**：利用已经计算好的回文中心和右边界信息，通过对称性直接“镜像拷贝”出当前位置的大致半径，从而大幅减少对比次数。

### 2. Manacher 算法的步骤

#### 第一步：字符串预处理
为了统一奇数长度（如 `aba`）和偶数长度（如 `abba`）的回文处理，我们在每个字符之间插入一个特殊字符（如 `#`）：
*   `aba` -> `#a#b#a#`
*   `abba` -> `#a#b#b#a#`
这样处理后，所有回文的长度都变成了奇数。

#### 第二步：维护两个核心变量
*   `C` (Center)：目前已知回文的最右边界所对应的中心。
*   `R` (Right boundary)：目前已知回文的最右边界（`R = C + P[C]`）。
*   `P[i]`：以 `i` 点为中心的回文半径。

#### 第三步：核心逻辑
对于当前位置 `i`：
1.  **镜像复用**：找到 `i` 关于 `C` 的对称点 `i_mirror = 2*C - i`。如果 `i < R`，则 `P[i]` 至少为 `min(R - i, P[i_mirror])`。
2.  **尝试扩展**：以 `i` 为中心向外尝试扩展。
3.  **更新状态**：如果当前回文的右边界超过了 `R`，则更新 `C = i` 和 `R = i + P[i]`。

---

### 3. C++ 代码实现

```cpp
#include <iostream>
#include <vector>
#include <string>
#include <algorithm>

using namespace std;

int manacher(string s) {
    // 1. 预处理：将 s 转为 t，例如 aba -> #a#b#a#
    string t = "#";
    for (char c : s) {
        t += c;
        t += "#";
    }

    int n = t.length();
    vector<int> p(n, 0); // p[i] 记录以 i 为中心的回文半径
    int C = 0, R = 0;    // C 是当前最大回文的中心，R 是最右边界
    int max_len = 0;

    for (int i = 0; i < n; i++) {
        // 利用对称性初始化 p[i]
        if (i < R) {
            p[i] = min(R - i, p[2 * C - i]);
        }

        // 向外扩展
        while (i - p[i] - 1 >= 0 && i + p[i] + 1 < n && t[i - p[i] - 1] == t[i + p[i] + 1]) {
            p[i]++;
        }

        // 更新中心和右边界
        if (i + p[i] > R) {
            C = i;
            R = i + p[i];
        }

        // 记录最大值
        max_len = max(max_len, p[i]);
    }
    return max_len;
}

int main() {
    ios::sync_with_stdio(false); // 加速输入输出
    cin.tie(nullptr);

    string s;
    cin >> s;
    cout << manacher(s) << endl;
    return 0;
}
```

---

### 4. 关键点总结
1.  **时间复杂度分析**：虽然代码里有双重循环（`for` 外加 `while`），但 `R` 是单向递增的，每个位置最多只会被访问常数次，因此总时间复杂度是 **$O(n)$**。
2.  **空间复杂度**：需要开辟一个长度为 $2n+1$ 的数组来存储半径，因此空间复杂度为 **$O(n)$**。
3.  **为什么处理后长度刚好？** 因为预处理后的字符串中，回文半径 `p[i]` 的值恰好就是原始字符串中对应回文子串的长度。例如 `aba` 变成 `#a#b#a#` 后，中心 `b` 处的半径是 3，对应原字符串长度正好是 3。

这种算法是解决此类题目在 $O(n)$ 时间内的标准写法，建议熟记其基本实现思路。
