# Find Eventual Safe States/找到最终的安全状态
**一、题目再读（中文版）**

- 给定一个有向图，节点编号为 `0 … n‑1`。  
- `graph[i]` 中列出所有从节点 `i` 出发的直接后继（出边）。  
- **终端节点（terminal node）**：出度为 `0`（没有任何出边）的节点。  
- **安全节点（safe node）**：从该节点出发**所有可能的路径**最终都会到达一个终端节点。  
  换句话说，从安全节点出发的任何路径**永远不会被困在一个有向环里**。

**任务**：返回所有安全节点并按升序输出。

---

**二、输入/输出格式**

| 位置 | 内容 | 说明 |
|------|------|------|
| 第 1 行 | `n`（`1 ≤ n ≤ 10⁴`） | 节点数量 |
| 第 2 … n+1 行 | `d_i`（第 i 行的出度）后面跟 `d_i` 个整数，表示所有后继节点 `v` | `graph[i]`。保证每个后继列表是递增的，可能出现自环。<br>**总边数** `E` 满足 `1 ≤ E ≤ 4·10⁴`。 |
| 第 1 行（输出） | `m` | 安全节点的个数 |
| 第 2 行（输出） | `m` 个整数，空格分隔，升序 | 若 `m = 0`，输出空行（只换行） |

> 例子  
> 输入 1  
> ```
> 7
> 2 1 2
> 2 2 3
> 1 5
> 1 0
> 1 5
> 0
> 0
> ```
> 输出 1  
> ```
> 4
> 2 4 5 6
> ```

---

**三、涉及的核心知识点**

| 知识点 | 关联概念 |
|--------|----------|
| **有向图（Directed Graph）** | 节点、出边、后继、出度 |
| **环（Cycle）** | 只要图中存在从节点出发能回到自身的环，就不可能是安全节点 |
| **拓扑排序（Topological Sort）** | 通过逆向过程把终端节点当作“入度为 0”的点，逐层删除，最终留下的节点必然在环中 |
| **逆向图（Reverse Graph）** | 用逆向边把“**从后继出发**”的问题转化为“**从终端出发**”的问题，便于拓扑删除 |
| **DFS/状态机** | 用 3 种颜色（未访问/正在访问/安全）递归检查是否在环中，也是常见的解法 |
| **Kahn 算法**（基于入度的 BFS） | 这里的“入度”其实是“逆向图中的入度”，即每个节点在原图中被多少条出边指向（后继数量） |

> 本题最直观的两种思路：**DFS+三色标记** 与 **逆向拓扑（Kahn）**。这里我们重点展示 **逆向拓扑**，因为它的时间/空间复杂度更稳定，易于实现。

---

**四、解题思路（逆向拓扑 + Kahn）**

### 1. 关键观察

- **终端节点**（出度 `0`）显然是安全的，因为从它出发已经没有任何路径可以继续走。  
- 如果一个节点的所有后继都已经被判定为安全，那么该节点本身也一定是安全的。  
- 反之，**只要一个节点存在至少一条指向非安全节点（即仍在环中的节点）的出边**，它就不安全。

> 于是我们把“**安全**”从终端节点逐步向外推广，这正好对应 **拓扑排序的逆向过程**。

### 2. 逆向图与入度

- 建立 **逆邻接表** `revAdj[j]`，保存所有指向节点 `j` 的前驱（即在原图中 `i → j`，则 `revAdj[j]` 包含 `i`）。  
- 计算每个节点的 **出度** `outDeg[i]`（即原图中的后继个数），这恰好等于在逆图中的 **入度**（`revInDeg[i]`）。  
- 将所有 **outDeg[i] == 0** 的节点（终端）加入队列 `q`。

### 3. Kahn 过程（从终端向外“消环”）

```
while q not empty:
    v = q.pop()
    // v 已经确定是安全的
    for each u in revAdj[v]:   // u 是原图中指向 v 的节点
        outDeg[u]--            // 相当于把边 u -> v 从图中删掉
        if outDeg[u] == 0:     // u 失去所有“未安全”后继，变成安全
            q.push(u)
```

- 当队列 `q` 弹出的节点集合即为 **所有安全节点**（因为它们要么是终端，要么所有后继都已安全）。  
- 剩下的未被弹出的节点必然属于 **环**（或其所在的不可达环），因此不是安全节点。

### 4. 正确性说明（简要）

- **命题 1**：所有终端节点（outDeg = 0）安全。  
- **命题 2**：如果一个节点 `u` 的所有后继都已安全，则 `u` 必然安全。  
- **逆命题 2 的逆否**：如果 `u` 不是安全节点，则必然存在至少一个后继 `v` 不是安全节点。  
- Kahn 过程正是通过**不断删除已经被确定为安全的出边**来模拟上述命题的递推。  
- 当过程结束后，只有 **无法被删除**（即仍有未安全后继）的节点 остаются，它们必然在环中或可到达环。  
- 因此，**被删除的节点集合恰好是安全节点的集合**。

### 5. 复杂度分析

| 项目 | 时间 | 空间 |
|------|------|------|
| 构建逆邻接表 | O(E) | O(E) |
| Kahn 主循环（每条边恰好被遍历一次） | O(E) | O(n)（队列 + outDeg + revAdj） |
| **总计** | **O(n + E)** | **O(n + E)** |

这满足题目限制（`n ≤ 10⁴, E ≤ 4·10⁴`）。

**五、C++ 实现（完整代码）**

```cpp
#include <bits/stdc++.h>
using namespace std;

/*
 * 题目：找到所有“安全节点”（LeetCode 802: Find Eventual Safe States）
 * 思路：逆向拓扑 + Kahn（从终端节点向前提取安全节点）
 *
 * 1. 读取 n 和每条边的出度、后继列表。
 * 2. 构建逆邻接表 revAdj（存储所有指向当前节点的前驱）。
 * 3. 统计每个节点的出度 outDeg（即原图的出度）。
 * 4. 把所有 outDeg == 0 的终端节点放入队列 q。
 * 5. BFS 过程：弹出节点 v，遍历其所有前驱 u，outDeg[u]--，
 *    若 outDeg[u] 变为 0，则 u 也是安全的，加入队列。
 * 6. 最终被访问过的节点即为安全节点，收集并排序输出。
 */

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int n;
    if (!(cin >> n)) return 0;   // 防止空输入
    vector<vector<int>> revAdj(n);   // 逆邻接表：revAdj[v] = 所有 u 使得 u -> v
    vector<int> outDeg(n, 0);        // 出度（在逆图中等价于入度）
    
    for (int i = 0; i < n; ++i) {
        int d;
        cin >> d;
        outDeg[i] = d;
        for (int k = 0; k < d; ++k) {
            int v;
            cin >> v;
            // 原图中 i -> v，则在逆图中 v -> i
            revAdj[v].push_back(i);
        }
    }
    
    // Kahn 过程
    queue<int> q;
    for (int i = 0; i < n; ++i) {
        if (outDeg[i] == 0) q.push(i);   // 终端节点必定安全
    }
    
    vector<char> isSafe(n, 0);   // 0/1 标记
    while (!q.empty()) {
        int v = q.front(); q.pop();
        isSafe[v] = 1;   // v 已经确定安全
        
        for (int u : revAdj[v]) {   // 所有指向 v 的前驱
            // 删除边 u -> v（即把 v 从 u 的后继集合中抹去）
            outDeg[u]--;
            if (outDeg[u] == 0) {   // u 的所有后继都已安全
                q.push(u);
            }
        }
    }
    
    // 收集安全节点并排序（题目要求升序）
    vector<int> ans;
    for (int i = 0; i < n; ++i)
        if (isSafe[i]) ans.push_back(i);
    
    // 输出
    cout << ans.size() << "\n";
    if (!ans.empty()) {
        for (size_t i = 0; i < ans.size(); ++i) {
            if (i) cout << ' ';
            cout << ans[i];
        }
    }
    cout << "\n";
    return 0;
}
```

### 代码说明

| 代码段 | 作用 |
|--------|------|
| `revAdj` | 逆邻接表，便于在删除一条边时快速定位所有受影响的**前驱**（原图的**后继**）。 |
| `outDeg` | 每个节点的出度。初始化即读取的 `d_i`。在 Kahn 循环中它充当**逆图的入度**，每删一条边就减 1。 |
| `queue<int> q` | 存放已经确认安全的节点（终端或所有后继已安全的节点）。 |
| `isSafe` | 标记哪些节点最终被弹出队列，即安全节点。 |
| `ans` | 收集并排序后输出。 |

### 边界情况

1. **全是环**：没有任何 `outDeg == 0` 的节点，队列空，答案 `0`。
2. **自环**：如果节点 `i` 有自环，则 `outDeg[i] ≥ 1`，即使它没有其他后继，它永远不会被加入队列，因为 `outDeg[i]` 不会降到 0。  
   这符合题意——自环本身形成了一个环，`i` 不是安全节点。
3. **空图**（`n = 1, d_0 = 0`）：唯一的节点既是终端节点也是安全节点，答案输出 `1` 随后 `0`。
4. **大图**：`n ≤ 10⁴, E ≤ 4·10⁴`，代码的时间/空间均在题目限制之内。

**七、如果你更喜欢 DFS 方式（简要思路）**

- 为每个节点维护三种状态：`0 = 未访问`，`1 = 正在访问（递归栈中）`，`2 = 已判定安全`。  
- 对每个未访问的节点执行 DFS，若在递归过程中再次遇到 `1`，说明发现环，标记为不安全。  
- 当递归返回时，若所有后继均为安全，则该节点安全（状态 `2`），否则不安全（状态 `1` 或者 `0`）。  
- 复杂度同样 `O(n+E)`，但递归深度可能达到 `n`（需要增加 `vector<char>` 栈或改写为显式栈）。

**八、结语**

- 本题的核心是 **“把所有能到达终端节点的节点挑出来”**，这正是 **逆向拓扑**（从终端向外删除）能自然完成的工作。  
- 关键在于构造逆邻接表、记录出度、以及一次 BFS（Kahn）过程。  
- 代码实现简洁，易于调试，且满足题目所有约束。
