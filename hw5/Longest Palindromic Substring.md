### 1. 题意理解 (核心点)
*   **输入**：一个字符串 $S$。
*   **输出**：最长回文子串的长度。
*   **回文特征**：正反读都一样。
*   **矛盾焦点**：
    *   **$N \le 1000$**：这是 **“友好”的题**。$O(n^2)$ 的算法够用，代码简单易懂，不易出错。
    *   **$N \ge 10^5$**：这是 **“苛刻”的题**。$O(n^2)$ 会超时（TLE），必须使用 Manacher 算法（$O(n)$）。

---

### 2. 涉及知识点 (分层对比)

| 特性 | 基础方案 (中心扩展法) | 进阶方案 (Manacher 算法) |
| :--- | :--- | :--- |
| **适用规模** | $N \le 2,000$ | $N \le 100,000+$ (及以上) |
| **核心复杂度** | $O(n^2)$ | $O(n)$ |
| **核心思想** | 遍历所有可能的“中心”，向外扩散比较。 | 利用回文的对称性，缓存已知信息，避免重复比较。 |
| **预处理** | 无 (需要分别处理奇/偶情况) | 必须 (插入 `#` 将奇偶统一) |

### 3. 代码细节纠正与实战

#### 方案 A：中心扩展法 ($O(n^2)$)
**适用场景**：面试中非高频优化题，或者是 $N \le 1000$ 的简单题。
*   **纠正细节**：
    *   **奇偶统一处理**：不要试图在同一个函数里搞定奇偶，直接分别写 `expandAroundCenter(s, i, i)` 和 `expandAroundCenter(s, i, i+1)` 最清晰。
    *   **长度返回**：循环停止时 `left` 和 `right` 越界了，公式一定要记牢：`right - left - 1`。

```cpp
// 核心逻辑片段
int expand(const string& s, int l, int r) {
    while (l >= 0 && r < s.size() && s[l] == s[r]) {
        l--; r++;
    }
    return r - l - 1; // 关键点：返回实际长度
}

// 遍历每个点
for(int i=0; i<n; ++i) {
    max_len = max({max_len, expand(s, i, i), expand(s, i, i+1)});
}
```

---

#### 方案 B：Manacher 算法 ($O(n)$)
**适用场景**：明确要求 $O(n)$ 时间复杂度，或者是 $N$ 很大。
*   **纠正细节（新手最易错的地方）**：
    *   **边界保护**：在 `while` 扩展时，`t[i - p[i] - 1] == t[i + p[i] + 1]` 前面一定要先加上边界判断 `i - p[i] - 1 >= 0 && i + p[i] + 1 < t.size()`，否则会发生越界堆栈溢出。
    *   **数组定义**：`p` 数组的大小应该是 `2*n + 1`，预处理后的字符串长度是原始的 2 倍多。

```cpp
// 转换字符串，例如 aba -> #a#b#a#
string T = "#";
for(char c : s) { T += c; T += "#"; }

// 核心扩展：p[i] 为当前中心的回文半径
// 这里的技巧是 p[i] 的值刚好就是该回文在原字符串中的长度
while(i - p[i] - 1 >= 0 && i + p[i] + 1 < n && T[i - p[i] - 1] == T[i + p[i] + 1]) {
    p[i]++;
}
```

### 总结：你的“决策树”

下次看到这道题，先看一眼数据范围 $N$：

1.  **一眼过 $N$**：
    *   如果 $N \le 1000$，**闭着眼睛用“中心扩展法”**。代码量少，不容易写 bug，面试现场写得出来就是胜利。
    *   如果 $N > 10^5$，**启用 Manacher 算法**。这是考察基本功的“天花板”算法。

2.  **代码结构建议**：
    *   建议在你的代码库中建立两个模板：`LongestPalindrome_Basic.cpp` 和 `LongestPalindrome_Manacher.cpp`。
    *   Manacher 算法不需要为了考研或者面试去背诵细节（除非是专门考字符串），理解 **“利用对称性减少比较次数”** 这个核心思想即可。

这个梳理方式把“不同场景对应不同代码”分开，这样你在复习时就不会觉得它们是一团乱麻了！
