力扣上的“跳跃游戏”系列虽然名字都叫跳跃，但根据问题的本质，可以清晰地划分为四大技术领域。**前两题是贪心，后面这几题则完全步入了图论、DP 和搜索的高级范畴。**

以下是这六道题的归纳总结：

---

### 1. 广度优先搜索 (BFS) 系列
**特征：** 要求“最少跳跃次数”或“能否到达”，且图的边权重为 1。

*   **跳跃游戏 III：** 相当于在一个**无权图**中寻找从起点出发是否能到达目标节点（0）。使用 BFS 或 DFS 遍历即可。
*   **跳跃游戏 IV：** 核心难点在于**建立图的边**。由于相同数值的下标可以互跳，必须先用 `unordered_map<int, vector<int>>` 将相同数值的下标建立邻接关系，避免 $O(N^2)$ 的建图复杂度，之后就是一个标准的**最短路径 BFS**。

### 2. 动态规划 (DP) 系列
**特征：** 要求“最大得分”或在满足特定限制下的路径最优解。

*   **跳跃游戏 VI：** 求最大得分。方程为 `dp[i] = nums[i] + max(dp[i-k...i-1])`。因为数据范围达到 $10^5$，普通的 DP 会超时（$O(N \cdot K)$），需要使用 **单调队列 (Deque)** 或者 **优先队列** 来优化寻找区间最大值的过程，将时间复杂度降至 $O(N)$。
*   **跳跃游戏 VII：** 虽然看起来像 BFS，但为了处理 $O(N)$ 复杂度的范围跳跃，通常结合 **前缀和** 或 **滑动窗口** 来优化状态转移，确保能在 $O(N)$ 时间内判断当前位置是否可达。

### 3. 记忆化搜索 / 深度优先搜索 (DFS) 系列
**特征：** 每个点的跳跃规则极具方向性和限制性，类似在有向无环图 (DAG) 上寻找最长路径。

*   **跳跃游戏 V：** 这是一个典型的 **带限制的 DAG 路径问题**。由于只能向数值较小的方向跳且距离有限制，每个点可到达的路径是确定的。使用 **记忆化搜索 (Memoization)** 存储每个点作为起点的“最长可访问下标数”，不仅避免重复计算，还能处理复杂的跳跃限制。

### 4. 动态规划 + 贪心/排序思想 (进阶题目)
*   **跳跃游戏 IX：** 这道题看起来很像图论，但由于其跳跃的方向性（先小后大，或先大后小），实质上是在考察**如何构建路径以获得全局最大值**。这通常涉及对数组元素进行排序或建立索引关系，利用单调性来推导每个位置能触达的“最大值”。

---

### 总结：知识点匹配图

| 题号 | 题目描述 | 核心算法/数据结构 | 解决关键点 |
| :--- | :--- | :--- | :--- |
| **I, II** | 基础跳跃 | **贪心算法** | 维护可达边界 maxReach |
| **III** | 到达目标坐标 | **BFS / DFS** | 简单的图遍历 |
| **IV** | 最少跳跃次数 | **BFS + 哈希表** | 同值下标建边，避免超时 |
| **V** | 最多访问点数 | **记忆化搜索 (DFS)** | 在限制条件下挖掘 DAG 性质 |
| **VI** | 最大得分 | **DP + 单调队列** | 区间最大值优化 |
| **VII** | 范围可达性 | **DP + 滑动窗口/前缀和** | 快速判断区间是否存在可达点 |
| **IX** | 到达最大值 | **贪心 / 排序 / 预处理** | 维护跳跃序列的单调性 |

---

### 学习建议：
1.  **区分“步数”和“得分”：**
    *   问**步数**：优先考虑 BFS；如果步数有范围限制，考虑 DP。
    *   问**得分**：一定是 DP，关键在于如何用单调队列/堆优化 DP 的转移过程。
2.  **处理“等值跳跃”：** 一旦题目出现“值相等可以跳”，一定要想到用 `Map` 存索引，这是将问题转化为图论的最通用手段。
3.  **处理“距离限制”：** 如果限制跳跃距离 `d`，要思考是用滑动窗口还是单调队列；如果限制高度关系，要思考是否构成了 DAG，从而使用记忆化搜索。

这六道题把“跳跃”的外壳剥掉后，其实就是**图论遍历（BFS/DFS）**与**线性动态规划（DP+优化）**的综合测试，掌握了这些套路，你就掌握了线性结构搜索题的核心。
