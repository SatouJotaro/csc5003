**一、题目描述（中文）**

- 给你一个 `R × C` 的二维网格（`R`、`C` 最多 1000，且 `R·C ≤ 5·10⁵`）。  
- 每个格子只能是三种值之一  

| 值 | 含义 |
|---|------|
| 0 | 空格（没有橘子） |
| 1 | 新鲜的橘子 |
| 2 | 已经烂掉的橘子 |

- 每一分钟，所有与**已经烂掉的橘子**相邻（上下左右四个方向）的新鲜橘子会立刻变成烂橘子。  
- 要求算出把所有新鲜橘子都变成烂橘子所需的**最少分钟数**。  
  - 若根本不可能全部变烂，输出 `-1`。  
  - 若网格里一开始就没有新鲜橘子（只有 0 和 2），则答案为 `0`。  

**输入格式**  

```
R C
grid[0][0] grid[0][1] … grid[0][C-1]
grid[1][0] …                     …
…
grid[R-1][0] … grid[R-1][C-1]
```

每行有 `C` 个整数，之间用空格分隔（换行或多余空格均可）。  

**输出格式**  

一个整数，表示最少需要的分钟数，或 `-1`（不可能的情况）。

---

**二、涉及的知识点**

- **图的遍历（宽度优先搜索 BFS）**  
  - 每个橘子可以看成一个结点，若两个橘子相邻（上下左右），它们之间有一条边。  
  - 这里的“距离”定义为分钟数，所有边的权值都是 1，故从所有已经烂掉的橘子（多源）出发进行 BFS，能够在最早的时刻把每颗新鲜橘子遍历到。  
- **多源 BFS**：一次性把所有的起始点（所有 2）放进队列，随后逐层扩展。  
- **网格的坐标压缩**、**方向数组**的常规技巧。

---

**三、解题思路（详细步骤）**

1. **读取输入**  
   - 用 `vector<vector<int>> grid` 保存网格。  
2. **初始化队列**  
   - 遍历整个网格：  
     - 若是 `2`（已经烂），把它放进队列 `q`，并在队列中携带它的“时间”信息（或在 BFS 循环中记录层数）。  
     - 若是 `1`（新鲜），统计新鲜橘子总数 `freshCnt`。  
   - 若 `freshCnt == 0`，直接输出 `0` 并结束。  
3. **多源 BFS**  
   - 记录当前层的结点数 `levelSize = q.size()`，这恰好是本分钟开始时已经烂掉的橘子数量。  
   - 使用 `minutes = 0` 记录已经过去的时间。  
   - 循环遍历队列：  
     - 当 `levelSize == 0` 时，说明本层的所有橘子已经处理完，`minutes++`，并更新 `levelSize = q.size()`（即下一层的橘子数）。  
     - 否则弹出一个烂橘子 `(r,c)`，检查四个相邻格子：  
       - 若相邻格子在网格内且值为 `1`（新鲜），把它改成 `2`（烂），`freshCnt--`，并把该格子加入队列（它的到达时间就是 `minutes+1`，在层数上体现为进入下一层）。  
4. **结束条件**  
   - BFS 结束后，如果 `freshCnt > 0`（仍有新鲜橘子），说明有些橘子永远不可能被感染，输出 `-1`。  
   - 否则返回 `minutes`（即最后一分钟结束时的层数）。  

**关键细节**  

- **层数的实现**：可以采用「记录当前层的 size」的方式，每处理完一整层就 `minutes++`。  
- **时间复杂度**：每个格子最多进入队列一次，故 `O(R·C)`。  
- **空间复杂度**：队列最多存放所有格子，故 `O(R·C)`（在实际实现中只保存坐标即可）。

**四、C++ 实现（使用 C++17）**

```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int R, C;
    if (!(cin >> R >> C)) return 0;        // 防止空输入
    vector<vector<int>> grid(R, vector<int>(C));
    
    queue<pair<int,int>> q;                // 存放已经烂掉的橘子坐标
    int freshCnt = 0;
    
    for (int i = 0; i < R; ++i) {
        for (int j = 0; j < C; ++j) {
            cin >> grid[i][j];
            if (grid[i][j] == 2) {
                q.emplace(i, j);           // 初始烂橘子入队
            } else if (grid[i][j] == 1) {
                ++freshCnt;
            }
        }
    }
    
    if (freshCnt == 0) {                   // 没有新鲜橘子
        cout << 0 << '\n';
        return 0;
    }
    
    const int dr[4] = {-1, 1, 0, 0};
    const int dc[4] = {0, 0, -1, 1};
    
    int minutes = 0;
    while (!q.empty()) {
        int levelSize = (int)q.size();    // 本分钟开始时的烂橘子数
        // 处理本层的所有橘子
        for (int k = 0; k < levelSize; ++k) {
            auto [r, c] = q.front(); q.pop();
            for (int d = 0; d < 4; ++d) {
                int nr = r + dr[d];
                int nc = c + dc[d];
                if (nr < 0 || nr >= R || nc < 0 || nc >= C) continue;
                if (grid[nr][nc] == 1) {
                    grid[nr][nc] = 2;      // 感染
                    --freshCnt;
                    q.emplace(nr, nc);    // 进入下一层的队列
                }
            }
        }
        ++minutes;                        // 本层处理完后过去一分钟
    }
    
    if (freshCnt > 0) cout << -1 << '\n';
    else               cout << minutes-1 << '\n'; // minutes 多算了一次，因为最后一次层结束后又加 1
    return 0;
}
```

**说明**  

- `minutes` 在每处理完一整层后加 1。BFS 结束时 `minutes` 比实际所需多 1（例如全部橘子最初已经是烂的，循环一次也不进入，minutes 为 0）。因此在输出时如果 `freshCnt==0`，我们返回 `minutes-1`。  
- 若在 BFS 结束后仍有新鲜橘子，说明这些橘子永远无法被感染，输出 `-1`。  

---

**五、复杂度分析**

| 项目 | 复杂度 |
|------|--------|
| 时间 | `O(R·C)`（每个格子最多入队一次） |
| 空间 | `O(R·C)`（队列最坏情况下存放全部格子） |

---

**六、总结**

- 这道题本质上是 **多源最短路径**（无权图的 BFS）问题。  
- 通过一次性把所有已经烂掉的橘子作为起点，同时向四周扩展，即可得到每个新鲜橘子最早被感染的时间。  
- 当且仅当所有新鲜橘子都被遍历到时，才存在可行的答案；否则返回 `-1`。
