### 一、 题意理解：多源同步传播
*   **核心逻辑**：这是一个“扩散”过程。不是某一个烂橘子在动，而是**所有烂橘子同步地向四周蔓延**。
*   **状态空间**：
    *   `0`：背景（无影响）。
    *   `1`：目标（待处理）。
    *   `2`：源点（产生影响的起点）。
*   **目标**：寻找“最后一次感染动作”发生的时间。
*   **关键处理逻辑**：如果在 BFS 结束后依然存在 `1`，说明这些格子被“与世隔绝”了，无法到达。

### 二、 涉及知识点：图的广度优先搜索 (BFS)
1.  **多源 BFS (Multi-source BFS)**：
    *   普通的 BFS 是从一个点出发，而多源 BFS 是将所有起点同时入队。这种方式天然地计算出了每个新鲜橘子到“最近的烂橘子”的最短距离。
2.  **层序遍历 (Level-order Traversal)**：
    *   本题的核心在于“分钟”的统计。通过“固定每一层的队列长度 `levelSize` ”，可以将 BFS 的每一次外层迭代形象地对应为“过去了一分钟”。
3.  **邻接表/坐标偏移**：
    *   使用方向数组 `dr[]` 和 `dc[]` 简化对四个邻居（上下左右）的访问逻辑，避免复杂的 if-else 嵌套。

### 三、 代码细节纠正与优化建议
在你提供的代码中，逻辑整体是正确的，但有两处关键细节建议注意：

#### 1. 关于 `minutes` 的计算校准
你目前的逻辑是 `minutes` 在循环底部 `++`，最后输出 `minutes - 1`。这在逻辑上是通的，但略显别扭。
**优化写法**：可以将判断逻辑前置，使代码更符合直觉。

```cpp
int minutes = 0;
// 如果一开始就有新鲜橘子才会进入循环
while (!q.empty() && freshCnt > 0) {
    int levelSize = q.size();
    for (int k = 0; k < levelSize; ++k) {
        // ...执行感染逻辑...
        if (grid[nr][nc] == 1) { /*...*/ }
    }
    minutes++; // 只有在确实发生了感染（或者还要继续处理）时才增加分钟数
}
```

#### 2. 关于 `minutes` 为 0 的边界处理
*   **存在 `1` 的情况**：如果 `freshCnt > 0`，但 BFS 结束后仍有残余，输出 `-1`。
*   **若一开始全是烂橘子**：你的逻辑中通过 `if (freshCnt == 0)` 提前处理了 `0` 的输出，这是非常好的，避免了 BFS 的冗余计算，**这一步不可省略**。

#### 3. 性能优化建议
*   **空间冗余**：当前的 `vector<vector<int>> grid` 空间占用为 `O(R*C)`，对于 `R*C = 5*10^5` 的数据量，内存消耗约 2MB，在常见的 256MB 限制下非常安全。
*   **输入优化**：题目说明输入可能含有多余空格，`cin` 对空格和换行是自动忽略的，所以你的 `cin` 写法是鲁棒的。
