# LeetCode 55：跳跃游戏 I（可行性问题）

### 1. 题意理解
*   **目标：** 判断是否能到达最后一个下标。
*   **输入：** 数组 `nums`，每个下标处的数字代表“在该位置能跳的最大步数”。
*   **关键点：** 这不是一个确定的路径规划，而是**范围扩展**。不需要关心怎么跳，只关心“能不能够着”。

### 2. 涉及知识点
*   **贪心算法 (Greedy Strategy)：** 维护一个“最远可到达的边界”。只要你在移动过程中，能够覆盖到的边界始终大于等于当前位置，你就一定能继续走下去。

### 3. 代码细节纠正
*   **边界陷阱（失败）：** 如果在遍历过程中，`i > maxReach`，说明当前位置 `i` 都走不到，后面就更别提了，立即返回 `false`。
*   **提前退出（优化）：** 一旦 `maxReach >= n - 1`，说明终点已经被覆盖，不需要继续遍历了，直接返回 `true`。
*   **关键点提醒：** 不要被 `nums[i]` 的值迷惑，它只是一个“贡献度”，是 `i + nums[i]` 决定了终点方向的延伸。

# LeetCode 45：跳跃游戏 II（最少步数问题）

### 1. 题意理解
*   **目标：** 到达最后一个下标的**最少**跳跃次数（题目保证一定能到达）。
*   **直觉转换：** 将数组切分成“一跳”一个区间。
    *   第一跳：从起点出发能覆盖的最远距离。
    *   第二跳：在第一跳覆盖范围内，能延伸到的最远距离。
    *   ...以此类推，像波浪一样推进。

### 2. 涉及知识点
*   **BFS/贪心结合：** 本质上是 BFS 的分层思想。每一跳相当于一层，贪心策略在于：**对于当前的跳跃范围，我一定要选一个能让下一次跳得最远的位置。**

### 3. 代码细节纠正
*   **逻辑纠正：** 这里有两个关键变量：
    *   `curEnd`：当前这一次跳跃所能到达的最远边界（也就是当前层的结束点）。
    *   `maxReach`：当前这一跳的所有选择中，能跳到的最远点（会更新为下一跳的边界）。
*   **循环结束点（重要）：**
    *   为什么循环 `i < n - 1`？因为当你站在最后一位时，不需要再跳了。如果执行到 `i == n-1` 可能会导致多跳一次。
*   **更新条件：** 当 `i == curEnd` 时，说明当前这一层看完了，必须增加跳次，并更新 `curEnd = maxReach`。

# 💡 总结对比：两题本质区别

| 对比项 | 55. 跳跃游戏 I | 45. 跳跃游戏 II |
| :--- | :--- | :--- |
| **考核点** | 可达性判断 | 路径最短化 (计数) |
| **核心逻辑** | **维护最远边界** (`maxReach`) | **分段推进** (BFS分层 + 贪心) |
| **思维瓶颈** | 怎么跳才走得通？(只要能走通就行) | 怎么跳才能步数最少？(每一步选能跳最远的) |
| **代码状态** | 维护一个 `maxReach` 即可 | 维护 `curEnd` 和 `maxReach` 两个变量 |
| **结局** | 只要 `maxReach` 到终点就赢 | 每次 `i` 走到区间边界就跳一次 |

### ⚠️ 一句话防混淆口诀：

*   **做第一题时（能否到达）：** 脑子里只要想着**“我能走到的最大范围能不能盖住终点”**，只要中间没断，就一定能到。
*   **做第二题时（最少几步）：** 脑子里想着**“我现在这站能决定下一站跳多远”**，一定要在当前这一站覆盖的范围内，替下一站找到它能跳到的最远位置。
