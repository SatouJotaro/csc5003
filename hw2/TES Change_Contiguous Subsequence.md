这是一个非常好的进阶问题。从“子序列（Subsequence）”变为“连续子串（Substring/Contiguous Subsequence）”，问题的本质从 **“贪心搜索”** 变为了 **“模式匹配”**。

在连续子串的问题中，元素不仅要出现，还得**贴在一起（连续出现）**。

---

### 一、题意理解：怎么变了？

*   **子序列（原题）：** 比如 `A = [1, 2, 3, 4]`，查询 `B = [1, 3]`。
    *   选取 `A[0]` 和 `A[2]`，符合条件，`TAK`。
*   **连续子串（新题）：** 同样 `A = [1, 2, 3, 4]`，查询 `B = [1, 3]`。
    *   `1, 3` 在原序列中中间隔了一个 `2`，不连续，所以是 `NIE`。
    *   如果要 `TAK`，查询必须是 `[1, 2]`、`[2, 3]` 或 `[1, 2, 3]` 这种在 `A` 中是一块的片段。

**题目描述变化：**
判断序列 `B` 是否存在于 `A` 的某一个连续区间 `A[i ... i+m-1]` 中。

---

### 二、解决办法：KMP 与 字符串哈希

如果要求连续，我们不再需要记录“所有出现位置的列表”，而是要把 `B` 当作一个整体，在 `A` 中作为一个“模版”进行匹配。

#### 1. KMP (Knuth-Morris-Pratt 算法)
KMP 是处理模式匹配的经典算法。
*   **核心逻辑：** 当你在匹配 `A` 和 `B` 时，如果遇到不匹配的字符，普通的暴力算法会重头再来。但 KMP 利用 `B` 自身结构的对称性（预处理出 `next` 数组），计算出**“如果匹配失败，我能往回跳多少步”**，从而避免无效的重试。
*   **优势：** 确定性算法，时间复杂度 $O(N+M)$，永远不会出错。
*   **适用场景：** 处理复杂的模式匹配，或者需要保证绝对正确性的场景。

#### 2. 字符串哈希 (Rolling Hash / Rabin-Karp)
*   **核心逻辑：** 将序列视为一个大整数。给 `B` 计算一个哈希值（比如 `Hash(B) = (b₁*p⁰ + b₂*p¹ + ... + bₘ*pᵐ⁻¹) % Mod`）。然后在这个窗口在 `A` 上滑动，快速计算 `A` 中每个长度为 `m` 的子串的哈希值。如果哈希值相等，则认为匹配成功。
*   **优势：** 对于序列匹配非常快，编写简单。
*   **风险：** 存在“哈希冲突”（即两个不同的序列算出了同一个哈希值）。通常需要使用“双哈希（Double Hash）”来极大概率规避冲突。

---

### 三、示例对比

假设：
序列 `A = [1, 2, 1, 2, 3]`
查询 `B = [1, 2, 3]`

| 算法 | 原理 | 复杂度 |
| :--- | :--- | :--- |
| **暴力法** | 每一个位置都尝试匹配 `B`，失败就退回。 | $O(N \cdot M)$ |
| **KMP** | 失败时根据 `next` 数组跳过不必要的比较。 | $O(N + M)$ |
| **哈希** | 比较 `Hash(B)` 与 `Hash(A[i...i+m-1])`。 | $O(N + M)$ |

这两种算法就是解决此类问题的“倚天剑”和“屠龙刀”。

### 一、KMP 算法（Knuth-Morris-Pratt）

**核心原理：**
KMP 的精髓在于**“利用已知信息，减少无效比较”**。
当你匹配 `A` 和 `B` 时，如果不匹配，普通的算法会从失败位置的下一个位置重新开始。但 KMP 发现，在失败之前的那一部分已经匹配成功了，且这部分中存在某种“前缀等于后缀”的对称结构。

*   **`next` 数组（或 `pi` 表）：** 它是 KMP 的灵魂。`next[i]` 记录的是：`B` 的前缀长度 `i` 的子串中，**最长相等前后缀的长度**。
*   **用途：** 当 `A[cur]` 和 `B[match]` 匹配失败时，我们不需要让 `A` 指针回溯，只需要让 `B` 的指针回溯到 `next[match - 1]` 位置即可。

**模板代码（适用于整数序列和字符串）：**

```cpp
#include <vector>
using namespace std;

// step 1: 计算 next 数组
vector<int> computeNext(const vector<int>& p) {
    int m = p.size();
    vector<int> next(m);
    for (int i = 1, j = 0; i < m; i++) {
        while (j > 0 && p[i] != p[j]) j = next[j - 1];
        if (p[i] == p[j]) j++;
        next[i] = j;
    }
    return next;
}

// step 2: KMP 匹配
bool kmpSearch(const vector<int>& a, const vector<int>& b) {
    int n = a.size(), m = b.size();
    if (m == 0) return true;
    vector<int> next = computeNext(b);
    for (int i = 0, j = 0; i < n; i++) {
        while (j > 0 && a[i] != b[j]) j = next[j - 1]; // 失配时跳转
        if (a[i] == b[j]) j++;
        if (j == m) return true; // 匹配成功
    }
    return false;
}
```

### 二、字符串哈希（Rolling Hash / Rabin-Karp）

**核心原理：**
把“一串”数字（序列 `B`）当做一个“超大整数”来看待。
把这个“大整数”存进一个 `unsigned long long` 变量中，对比两个序列是否相等，就变成了对比两个数字是否相等（$O(1)$ 的时间复杂度）。

*   **滑动窗口：** 我们不重新计算每个子串的哈希值，而是通过“移出最左元素，移入最右元素”的方式，在 $O(1)$ 时间内更新当前窗口的哈希值。
*   **抗冲突：** 使用 `unsigned long long` 自然溢出（相当于对 $2^{64}$ 取模）。为了防止“哈希冲突”（即两个不同的序列算出的哈希值一样），通常使用 **双哈希**（使用两个不同的基数/模数）。

**模板代码：**

```cpp
#include <vector>
typedef unsigned long long ull;

// 建议：实际比赛中，可以使用不同的基数(base)或双哈希(两个不同的ull)来降低冲突风险
const ull B = 13331; // 基数

bool hashSearch(const vector<int>& a, const vector<int>& b) {
    int n = a.size(), m = b.size();
    if (m > n) return false;
    
    ull hashB = 0;
    ull powerM = 1; // 记录 B^{m-1} 的值
    for (int i = 0; i < m; i++) {
        hashB = hashB * B + b[i];
        if (i < m - 1) powerM *= B;
    }
    
    ull hashA = 0;
    // 先计算 A 的前 m 个元素的哈希值
    for (int i = 0; i < m; i++) hashA = hashA * B + a[i];
    
    if (hashA == hashB) return true;
    
    // 滑动窗口
    for (int i = m; i < n; i++) {
        // 移除最左边: hashA - (A[i-m] * B^(m-1))
        // 加上最右边: hashA * B + A[i]
        hashA = (hashA - a[i - m] * powerM) * B + a[i];
        if (hashA == hashB) return true;
    }
    return false;
}
```

### 三、深度总结：什么时候选什么？

| 特性 | KMP | 哈希 (Rolling Hash) |
| :--- | :--- | :--- |
| **匹配对象** | 字符串、整数数组 | 字符串、整数数组 |
| **复杂度** | $O(N+M)$ | $O(N+M)$ |
| **正确性** | **严格准确** | **概率正确** (极极低概率冲突) |
| **实现难度** | 中等 (有点难想) | 低 (手写简单) |
| **主要危险** | 边界条件写错，`next` 数组理解偏差 | **遭遇造数据的“毒瘤”测试点导致冲突** |

#### 给你的建议：

1.  **如果你是在打简单的算法题（或者不想背 KMP）：**
    优先选择 **哈希**。代码行数少，逻辑简单，出 bug 的概率小。遇到哈希冲突（罕见）时，使用 `Double Hash`（两个哈希合在一起）可以保证通过率 99.99%。

2.  **如果你是在进行正式比赛或面试：**
    如果时间允许，**优先推荐 KMP**。因为它在算法逻辑上是“确定性算法”，对于考官或评测机来说，它是完全无懈可击的。

3.  **避坑指南（Integer Hash）：**
    上面的哈希模板针对的是整数数组。如果你的数组里数字巨大（比如达到 $10^9$），或者有负数，直接使用上面的哈希可能会溢出过快导致冲突。此时可以先把数字映射到较小的范围，或者使用更大的模数（比如 $10^9+7$）。
